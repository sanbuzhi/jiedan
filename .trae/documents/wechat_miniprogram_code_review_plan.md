# 微信小程序代码自查计划

## 目标
根据微信小程序开发规范，全面自查 `step_all` 页面及相关代码，确保没有bug，不会再有报错。

## 自查清单（基于开发规范）

### 一、WXML 语法检查

#### 1.1 事件绑定检查
- [ ] 检查所有 `bindtap`、`bindinput` 等事件名是否与JS方法名完全一致（逐字符匹配）
- [ ] 检查 `data-*` 属性是否正确传递参数
- [ ] 检查事件方法是否在JS中定义

#### 1.2 数据绑定检查
- [ ] 检查 `{{}}` 中的表达式是否为简单运算（无 `filter`/`map` 等复杂操作）
- [ ] 检查 `wx:key` 是否使用字段名（无多余大括号）
- [ ] 检查属性值是否使用双引号包裹

#### 1.3 列表渲染检查
- [ ] 检查 `wx:for` 是否正确使用
- [ ] 检查 `wx:key` 是否存在且正确

### 二、JS 逻辑检查

#### 2.1 数据管理检查
- [ ] 检查所有数据修改是否使用 `this.setData()`
- [ ] 检查数组/对象更新是否使用新引用
- [ ] 检查嵌套数据更新是否正确

#### 2.2 方法定义检查
- [ ] 检查所有在WXML中绑定的方法是否在JS中定义
- [ ] 检查方法名是否与WXML中绑定的一致
- [ ] 检查 `this` 指向是否正确（使用箭头函数）

#### 2.3 工具函数检查
- [ ] 检查使用的工具函数是否在 `util.js` 中定义
- [ ] 检查工具函数是否已添加到 `module.exports`
- [ ] 检查工具函数导入路径是否正确

### 三、数据字段一致性检查

#### 3.1 字段名三处一致性
- [ ] WXML 显示字段名
- [ ] JS 保存数据字段名
- [ ] JS 验证数据字段名

#### 3.2 阶段数据检查
- [ ] 阶段1：`requirementDescription`、`roles`、`selectedFunctionIds` 等
- [ ] 阶段2：`visualStyle`（统一字段名）
- [ ] 阶段3：`quotation` 相关字段

### 四、API 调用检查

#### 4.1 工具函数调用
- [ ] `util.getApiBaseUrl()` 是否可用
- [ ] `util.checkLogin()` 是否正确使用
- [ ] 其他工具函数是否正确导入和使用

#### 4.2 网络请求检查
- [ ] API 地址是否正确拼接
- [ ] 请求参数是否正确传递
- [ ] 错误处理是否完善

### 五、页面生命周期检查

- [ ] `onLoad` 是否正确处理参数
- [ ] `onShow` 是否检查登录状态
- [ ] 数据恢复和保存是否正确

### 六、边界情况检查

- [ ] 空值处理
- [ ] 网络错误处理
- [ ] 用户未登录处理
- [ ] 数据验证失败处理

## 实施步骤

1. **读取开发规范文档** - 确认检查标准
2. **读取 step_all.js** - 检查JS逻辑
3. **读取 step_all.wxml** - 检查WXML语法
4. **读取 util.js** - 检查工具函数
5. **交叉验证** - WXML与JS对照检查
6. **修复问题** - 根据检查结果修复
7. **验证修复** - 确保修复正确

## 预期结果

- 所有事件名WXML与JS完全一致
- 所有数据字段名三处一致
- 所有工具函数已定义并导出
- 无复杂表达式在WXML中
- 所有数据修改使用 `setData`

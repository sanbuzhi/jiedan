# AI 开发强制性规范

## 1. 路径计算原则
- 相对路径层级 = 当前文件到目标文件的目录层级差
- 禁止凭感觉写路径，必须逐层计算

## 2. 变量声明原则
- 模块在文件顶部统一声明
- 禁止在函数内部重复 require 同一模块

## 3. 数据访问原则
- 访问对象属性前必须做空值保护
- 数组索引访问前必须检查长度

## 4. 异步处理原则
- 显示 loading 必须配套 hide loading
- Promise 必须处理 catch 分支

## 5. API 调用原则
- 使用封装好的 API 模块方法
- 禁止直接调用底层请求方法
- **API 路径规范**：`util.getApiBaseUrl()` 已包含 `/api/v1` 前缀，调用时只需传入相对路径（如 `/exchange-items`），禁止重复添加 `/api/v1`
- **错误示例**：`api.get('/api/v1/exchange-items')` 会导致 URL 重复为 `/api/v1/api/v1/exchange-items`
- **正确示例**：`api.get('/exchange-items')`

## 6. 后端连接原则
- 本地开发使用 localhost
- 禁止直接连接远程数据库

## 7. 认证检查原则
- 受限页面必须在 onShow 检查登录状态
- 未登录立即跳转，阻止后续逻辑执行

## 8. 数据校验原则
- 后端校验必须允许合理的空值
- 禁止强制要求非空字段
- 异常处理必须保持 SecurityContext

## 9. 调试原则
- 修改代码后必须清除缓存验证
- 后端问题必须开启 DEBUG 日志定位

## 10. 文档原则
- 每个错误必须记录到规范文档
- 规范必须包含问题场景和解决方案

## 11. API 路径设计原则
- 必须统一使用 `/v1/xxx` 格式
- 禁止在 Controller 中重复添加 `/api` 前缀
- 上下文路径由 `server.servlet.context-path` 统一配置

## 12. DTO 设计原则
- 必须包含前端可能发送的所有字段
- 新增字段必须同步更新 DTO 类
- 禁止遗漏嵌套对象定义

## 13. 日志原则
- 生产环境禁止过度日志
- 调试日志仅在排查问题时临时添加
- 修复后必须清理临时日志

## 14. 响应格式原则
- 后端必须返回统一格式的响应 `{code, message, data}`
- 成功时 code = 0
- 错误时必须返回有意义的 message
- 禁止直接返回裸数据对象

## 15. 实体类设计原则
- 新增字段必须同步更新数据库表结构
- 枚举类型字段必须使用 `@Enumerated` 注解
- JSON 字段类型必须与使用方保持一致

## 16. 前后端数据一致性原则
- 后端返回的嵌套对象必须保持完整结构
- 禁止返回 null 导致前端渲染失败
- 空值情况下应返回默认值或空对象
- 前端 WXML 中访问嵌套属性前必须做空值检查

## 17. 前后端字段命名一致性原则
- **必须统一命名规范**：后端返回驼峰命名（如 `referralCode`），前端也必须使用驼峰命名访问
- **禁止混用命名风格**：前端不能将驼峰命名改为下划线命名（如 `referral_code`）
- **错误示例**：后端返回 `referralCode`，前端使用 `userInfo.referral_code` 访问，导致数据无法显示
- **正确示例**：后端返回 `referralCode`，前端使用 `userInfo.referralCode` 访问
- **调试方法**：接口返回数据后，先打印日志确认字段名，再绑定到视图

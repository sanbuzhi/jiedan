# Feedback Shadow 盯梢系统 - 架构文档（极简版）

## 1. 系统概述

### 1.1 系统定位
Feedback Shadow 盯梢系统是AI开发流程的质量保障系统，通过对AI接口输出的实时检测和质量门禁，确保AI生成代码的可用性和规范性。

### 1.2 核心目标
- **质量拦截**: 在开发阶段拦截编译错误、运行错误
- **规范约束**: 强制代码包结构、命名规范、代码风格
- **自我修复**: 自动检测并修复AI输出中的问题
- **闭环管理**: 用户反馈 → AI修复 → 验证 → 通知

### 1.3 技术栈
- **后端**: Spring Boot 3.x, Java 21
- **前端**: Vue 3, Element Plus, ECharts
- **小程序**: 微信小程序原生框架
- **数据库**: MySQL 8.x, Flyway 迁移
- **AI服务**: OpenAI API, Claude API, 通义千问等

### 5.2 新旧流程对比

| 特性 | 原流程（增量式） | 新流程（极简版） |
|------|-----------------|-----------------|
| **步骤数** | 5阶段（需求→拆分→生成→修复→交付） | 4阶段（需求→生成→修复→交付） |
| **AI调用次数** | 多次（脚手架+每个任务） | 1次（直接生成完整项目） |
| **复杂度** | 高（任务调度、上下文管理） | 低（单一生成点） |
| **代码量** | ~2000行 | ~500行 |
| **适用项目** | 大型项目 | 中小型项目 |
| **Token消耗** | 分散多次 | 单次大量 |

## 1. 需求概述
### 1.1 项目背景
当前XX酒店（或中型连锁/单体精品酒店模板）采用Excel表格+纸质单据混合管理的模式，存在以下核心痛点：
- 房态同步滞后（前台、客房、销售数据更新需手动传递，常出现超售/漏订）
- 订单管理混乱（线上线下多渠道未统一，对账周期长、差错率高）
- 客户资料零散（无统一会员体系，无法精准营销）
- 财务统计效率低（需人工汇总各类单据，数据真实性难追溯）
- 物资管理粗放（布草、易耗品、食品库存预警缺失，浪费/断供问题频发）

为解决上述问题，提升酒店运营效率、客户满意度与盈利能力，拟开发一套**全渠道一体化、模块化、可扩展**的酒店管理系统。

### 1.2 项目目标
1. **实时房态管理**：实现前台、客房、销售三端房态秒级同步，支持批量房态操作，杜绝超售/漏订
2. **全渠道订单整合**：对接OTA（携程、美团、飞猪）、官网/小程序、线下POS预订，统一订单入口、分配、处理与核销
3. **精细化会员体系**：建立积分、储值、等级、权益四位一体的会员管理，支持自定义规则与精准营销触达
4. **自动化财务闭环**：实现订单、押金、消费、退款、发票全流程财务数据自动生成与核对，支持多维度报表导出
5. **智能化物资管控**：建立布草损耗、易耗品/食品库存台账，设置多级预警机制，支持采购需求自动生成
6. **可扩展模块化设计**：预留第三方系统（门锁系统、POS收银系统、停车场系统、能耗监测系统）接口，支持按需选配功能模块

### 1.3 适用范围
本系统适用于**客房数50-500间的精品单体酒店**或**同规模的区域连锁酒店单店/集团总部管理**，涵盖前厅、客房、财务、销售、采购、会员六大核心业务部门。

## 2. 功能模块清单
本系统采用**集团总部-单门店**两级架构（单体酒店可关闭集团模块），共分为12个一级模块，37个二级模块，92个三级功能点，具体如下：

### 2.1 基础信息管理（集团+单门店）
#### 2.1.1 组织架构管理
- 支持创建/编辑/删除集团、区域、单门店三级组织（单体默认单级）
- 支持设置各组织的负责人、联系方式、地址、营业时间等基本信息
- 支持集团管理员分配/回收各组织的模块权限与数据权限

#### 2.1.2 客房基础配置
- 支持创建/编辑/删除楼层、区域（如商务楼、行政楼、亲子区）
- 支持创建/编辑/删除房型（如标准双床、豪华大床、套房、亲子房），配置房型的基础信息（面积、床型、可住人数、窗户情况、WiFi情况、设施清单、高清图片/视频）
- 支持批量创建/编辑/删除房间号，绑定房型、楼层、区域、门锁编号、电话分机号
- 支持设置房间的状态类型（如净房、脏房、维修房、维修中、锁定房、自用房、预留房）

#### 2.1.3 商品/服务基础配置
- 支持创建/编辑/删除餐饮商品（早餐、中餐、下午茶、夜宵）、非餐饮商品（布草包、洗漱用品升级、伴手礼）、服务（接机送机、洗衣熨烫、会议室租赁、SPA）
- 配置商品/服务的SKU、单价、成本价、库存类型（库存商品/非库存服务）、所属部门、适用房型/人群、促销规则适用范围
- 支持上传商品/服务的高清图片/视频

#### 2.1.4 支付方式配置
- 支持创建/编辑/删除支付方式（如现金、微信支付、支付宝支付、银联卡、储值卡、积分抵扣、OTA代扣）
- 配置支付方式的手续费率、对账周期、所属账户、是否需要实名认证、是否支持退款

#### 2.1.5 系统参数配置
- 支持配置单门店的超售规则（如按房型超售比例、按房间数固定超售、某时间段禁止超售）
- 支持配置会员积分规则（如消费1元积1分、储值1000元赠200积分、生日双倍积分）、储值规则（如储值1000元赠100元、储值5000元赠700元）、会员等级规则（如累计消费/积分达到XX元/分升级到银卡/金卡/钻石卡）
- 支持配置房态自动转换规则（如客人退房后30分钟自动转为脏房、客房打扫完成并确认后自动转为净房、维修完成并验收后自动转为净房）
- 支持配置发票规则（如增值税普通发票/专用发票的开票规则、开票限额、抬头类型限制）
- 支持配置预警规则（如易耗品/食品库存低于安全库存预警、布草损耗率超过阈值预警、OTA拒单率超过阈值预警、会员生日/纪念日提醒）

### 2.2 前厅管理（单门店）
#### 2.2.1 预订管理
- 支持多渠道预订入口：
  - 线下预订：前台员工手动录入客人信息、房型、入住/退房时间、支付方式、特殊需求
  - 官网/小程序预订：客人自助完成预订、支付、修改/取消（需符合系统规则）
  - OTA预订：自动同步携程、美团、飞猪等主流OTA的订单、支付状态、房态变化，支持一键确认/拒单
- 支持预订操作：
  - 批量预订、批量修改/取消预订
  - 锁定房间/预留房间给特定客人/团队
  - 自动分配房间（根据客人特殊需求、楼层偏好、房型剩余房、会员等级自动匹配）、手动调整房间
  - 查看预订详情、历史预订记录、预订轨迹
- 支持预订验证：
  - 验证身份证信息（对接公安户籍系统，生成入住登记表单）
  - 验证会员信息（自动识别会员等级、权益、积分/储值余额）
  - 验证预订支付状态（未支付预订可发送短信/微信提醒、超过保留时间自动取消）

#### 2.2.2 入住管理
- 支持散客入住：
  - 快速入住（身份证一刷自动读取信息、匹配预订、分配房间、生成房卡/开门码）
  - 无预订入住（手动录入信息、分配房间、选择支付方式）
  - 会员入住（自动识别会员等级、赠送欢迎礼/积分、推荐升级房型）
  - 同住人管理（最多添加4位同住人，验证同住人身份证信息）
- 支持团队入住：
  - 批量分配房间、批量验证身份证信息、批量生成房卡/开门码
  - 团队账单拆分/合并（可按房间、按日期、按客人拆分/合并账单）
  - 团队押金统一收取/分开收取
- 支持入住操作：
  - 收押金（支持多种支付方式，打印押金收据）
  - 升级房型（支持免费升级/付费升级，记录升级原因）
  - 延住（支持散客/团队延住，自动调整房态、账单）
  - 换房（支持散客/团队换房，自动调整房态、账单、房卡/开门码）
  - 查看入住详情、历史入住记录、入住轨迹

#### 2.2.3 退房管理
- 支持散客退房：
  - 快速退房（一键查房、自动结算账单、打印发票/消费明细、退还押金）
  - 延迟退房（根据会员等级/酒店规则自动判断是否免费延迟，或手动设置付费延迟）
  - 预授权完成/撤销（对接银联/微信/支付宝预授权系统）
- 支持团队退房：
  - 批量查房、批量结算账单、批量打印发票/消费明细、批量退还押金
  - 团队账单最终确认/拆分支付
- 支持退房操作：
  - 查房记录录入（客房员工APP端同步提交查房结果，记录房间消费、物品损坏情况）
  - 房间消费录入（支持手动录入迷你吧、洗衣熨烫、餐饮挂账等消费）
  - 退款（支持多种支付方式原路退回，打印退款凭证）
  - 开具发票（支持增值税普通发票/专用发票，支持电子发票/纸质发票，电子发票自动发送到客人邮箱/微信）
  - 会员评价引导（退房后自动发送短信/微信评价链接，评价后赠送积分）
  - 查看退房详情、历史退房记录、退房轨迹

#### 2.2.4 房态管理
- 支持可视化房态图（按楼层、区域展示房间状态，颜色区分：净房-绿色、脏房-灰色、维修房-红色、维修中-橙色、锁定房-紫色、自用房-蓝色、预留房-黄色、在住-粉色）
- 支持批量房态操作（批量转为净房/脏房/维修房/锁定房/自用房）
- 支持房态历史查询（查询某房间的状态变更时间、操作人、变更原因）
- 支持房态异常提醒（如净房超过24小时未入住、脏房超过2小时未打扫）

#### 2.2.5 前台POS收银
- 支持商品/服务扫码/手动录入消费
- 支持多种支付方式混合支付
- 支持账单挂账（挂房间账、挂团队账、挂协议单位账）
- 支持账单调整（调整消费金额、退款金额，记录调整原因、操作人）
- 支持打印消费明细、POS收银小票
- 支持POS收银交接班管理（打印交接班报表、核对现金/储值卡/预授权等金额、交接备用金）

### 2.3 客房管理（单门店）
#### 2.3.1 任务管理
- 支持任务自动分配（根据客房员工的工作量、负责楼层自动分配打扫任务、维修跟进任务）
- 支持任务手动分配/调整
- 支持任务状态跟踪（待分配、已分配、进行中、已完成、已验收）
- 支持任务优先级设置（VIP客人房间优先、维修中房间优先、即将到店客人房间优先）
- 支持任务催办（前台/客房主管APP端一键催办任务）

#### 2.3.2 打扫管理
- 支持客房员工APP端查看待打扫任务、任务详情、客人特殊需求
- 支持客房员工APP端提交打扫完成结果、上传房间照片/视频
- 支持客房主管APP端验收打扫结果（验收通过/不通过，不通过需说明原因并退回重扫）
- 支持打扫记录查询（查询某房间的打扫时间、操作人、验收人、验收结果）
- 支持打扫工作量统计（统计某客房员工/楼层/时间段的打扫房间数、VIP房间数、重扫房间数）

#### 2.3.3 查房管理
- 支持前台退房时自动触发查房任务
- 支持客房员工APP端查看待查房任务、任务详情
- 支持客房员工APP端提交查房结果（无问题/有问题，有问题需记录房间消费、物品损坏情况、损坏物品照片/视频）
- 支持查房记录查询（查询某房间的查房时间、操作人、查房结果、消费/损坏金额）

#### 2.3.4 布草管理
- 支持布草库存台账管理（入库、出库、盘点、报损、调拨）
- 支持布草标签管理（绑定RFID标签，实现布草的全生命周期追踪）
- 支持布草损耗统计（统计某时间段的布草损耗率、损耗金额、损耗原因）
- 支持布草洗涤管理（对接洗涤公司系统，自动生成洗涤订单、跟踪洗涤进度、核对洗涤数量/费用）

#### 2.3.5 易耗品管理
- 支持易耗品库存台账管理（入库、出库、盘点、报损、调拨）
- 支持易耗品领用管理（按房间/楼层/员工领用，自动扣减库存）
- 支持易耗品预警管理（库存低于安全库存/最低库存时，自动发送预警短信/微信给采购/客房主管）
- 支持易耗品消耗统计（统计某房间/楼层/时间段的易耗品消耗数量、消耗金额）

### 2.4 财务管理（集团+单门店）
#### 2.4.1 订单财务管理
- 支持订单财务数据自动生成（订单金额、押金金额、消费金额、退款金额、优惠金额、OTA佣金金额）
- 支持订单财务数据核对（与OTA平台、POS收银系统、支付平台自动核对，生成差异报表）
- 支持订单财务数据追溯（查询某订单的财务轨迹、操作人、操作时间）

#### 2.4.2 押金管理
- 支持押金台账管理（收取、退还、预授权完成/撤销、转消费）
- 支持押金报表统计（统计某时间段的押金收取金额、退还金额、预授权金额、转消费金额、押金余额）
- 支持押金异常提醒（如押金余额不足、预授权到期未处理）

#### 2.4.3 应收账款管理
- 支持协议单位账管理（创建/编辑/删除协议单位，设置协议单位的信用额度、账期、联系人、联系方式）
- 支持挂账管理（挂房间账、挂团队账、挂协议单位账，自动计入应收账款）
- 支持应收账款核销管理（手动/自动核销应收账款，生成核销凭证）
- 支持应收账款报表统计（统计某协议单位/时间段的应收账款余额、账龄分析、坏账准备）
- 支持应收账款预警管理（信用额度不足、账期到期时，自动发送预警短信/微信给销售/财务主管）

#### 2.4.4 应付账款管理
- 支持供应商管理（创建/编辑/删除供应商，设置供应商的联系人、联系方式、账期）
- 支持采购订单管理（自动生成采购订单/手动创建采购订单，跟踪采购进度）
- 支持采购入库管理（核对采购订单与入库单，自动计入应付账款）
- 支持应付账款核销管理（手动/自动核销应付账款，生成核销凭证）
- 支持应付账款报表统计（统计某供应商/时间段的应付账款余额、账龄分析）

#### 2.4.5 发票管理
- 支持发票台账管理（开具、作废、冲红、领取）
- 支持发票数据自动生成（与订单、消费数据自动关联，无需手动录入）
- 支持发票统计报表（统计某时间段的发票开具金额、作废金额、冲红金额、增值税普通发票/专用发票开具数量）
- 支持发票异常提醒（如发票库存不足、开票限额已达）

#### 2.4.6 财务报表管理
- 支持单门店财务报表：
  - 日报表（营业收入、押金收入/支出、应收账款/应付账款变动、入住率、平均房价、RevPAR）
  - 周报表/月报表/季报表/年报表（同上，多维度对比分析）
  - 房型销售报表（统计某房型的入住率、平均房价、RevPAR、销售收入）
  - 渠道销售报表（统计某渠道的订单量、入住率、平均房价、RevPAR、销售收入、OTA佣金）
  - 会员消费报表（统计某会员等级/会员的消费金额、消费次数、消费偏好）
- 支持集团财务报表：
  - 汇总单门店财务报表
  - 区域对比分析报表
  - 门店排名报表
- 支持所有报表的Excel/PDF导出、自定义时间段/维度查询、图表展示（柱状图、折线图、饼图）

### 2.5 销售管理（集团+单门店）
#### 2.5.1 协议单位管理
- 支持创建/编辑/删除协议单位，设置协议单位的类型（政府机关、企业单位、旅行社）、信用额度、账期、联系人、联系方式、地址
- 支持设置协议单位的折扣规则（如房型折扣、餐饮折扣、服务折扣）
- 支持协议单位订单自动关联折扣
- 支持协议单位消费统计报表

#### 2.5.2 团队管理
- 支持创建/编辑/删除团队信息，设置团队的类型（旅游团、会议团、商务团）、负责人、联系方式、入住/退房时间、房间数量、房型、折扣规则、支付方式
- 支持团队订单批量处理
- 支持团队消费统计报表

#### 2.5.3 促销管理
- 支持创建/编辑/删除促销活动（如满减活动、折扣活动、赠品活动、积分翻倍活动、储值赠现活动）
- 支持设置促销活动的时间范围、适用人群（所有客人、会员、银卡及以上会员）、适用房型/商品/服务、适用渠道、限制条件（如每个客人限参与一次、每天限参与N次、最低消费XX元）
- 支持促销活动自动生效/失效
- 支持促销活动效果统计报表（统计某促销活动的参与人数、订单量、销售收入、成本投入、ROI）

#### 2.5.4 渠道管理
- 支持对接OTA平台（携程、美团、飞猪、同程旅行等主流OTA），实现订单自动同步、房态自动同步、价格自动同步
- 支持创建/编辑/删除自有渠道（官网、微信小程序、抖音小程序）
- 支持设置各渠道的价格规则（如OTA价格为门市价的9折、自有渠道为门市价的8.5折、会员在自有渠道为门市价的8折）
- 支持各渠道的销售数据统计报表

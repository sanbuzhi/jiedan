## 1. 项目技术规格
- **技术栈**：
  - 后端：Java 17 + Spring Boot 3.2.5 + MyBatis Plus 3.5.5 + Swagger 3.0.0 + MySQL 8.0.36 + Redis 7.2.4
  - 管理端前端：Vue 3.4.21 + Element Plus 2.6.3 + Vite 5.2.8 + ECharts 5.5.0 + xlsx 0.18.5 + jspdf-autotable 3.5.31
  - 服务器：CentOS 7.9 + Nginx 1.24.0 + Spring Boot内置Tomcat

- **代码生成规则**：
  - 依赖版本严格锁定在技术栈指定版本；
  - 所有请求参数使用JSR-303/JSR-380校验注解；
  - JWT Token有效期7天，过期时间戳误差≤5分钟；
  - 敏感数据（密码、手机号、API Secret）使用AES-256加密存储；
  - 雪花ID作为主键生成策略，workId=1，datacenterId=1；
  - MP自动填充创建时间、更新时间、创建人、更新人。
- **核心业务领域锚点**：美妆小店轻量级后端管理系统

## 5. 业务逻辑规则
### 5.1 核心业务流程
#### 5.1.1 美妆商品采购入库流程
1. 仓库管理员/有采购权限的员工登录系统，进入采购单创建页面；
2. 选择/临时填写供应商，添加采购商品明细，填写支付方式等参数；
3. 保存并提交生成状态为“待入库”的采购单；
4. 仓库管理员/有入库权限的员工登录系统，进入待入库采购单列表；
5. 选择采购单，逐一核对商品并填写入库明细；
6. 确认入库，更新采购单状态为“部分入库”或“已完成”，同步更新商品库存；
7. 若采购单全部入库且支付方式为应付账款，提示是否立即结算。

#### 5.1.2 美妆商品零售单创建流程
1. 收银员登录系统，进入零售单创建页面；
2. 可选输入会员信息，扫码枪/手动输入商品条码添加销售商品明细；
3. 调整销售单价（≥最低零售价）、折扣率（优先会员等级折扣），确认库存充足；
4. 自动计算销售总金额，可选抹零/抹分；
5. 选择支付方式，点击确认收款完成操作；
6. 同步更新商品库存、会员资产（若关联），生成零售单并支持打印小票。

### 5.2 数据校验规则
1. 商品条码：唯一，不能为空；
2. 商品价格：最低零售价≥成本价，建议零售价≥最低零售价；
3. 会员手机号：唯一，符合手机号格式；
4. 采购/销售/退货/调拨/盘点数量：必须为正整数；
5. 实盘数量与系统库存数量不一致时，必须填写差异备注。

## 6. 开发执行顺序
### 6.1 阶段划分
1. **数据库阶段**：创建所有数据库表、索引、约束；
2. **后端核心基础阶段**：配置项目结构、依赖、工具类、统一响应、异常处理、JWT认证、MP自动填充；
3. **后端业务模块阶段**：按优先级从高到低开发进销存→用户→系统设置→流水报表→通用前端对接接口；
4. **前端基础阶段**：配置项目结构、依赖、路由、状态管理、接口请求封装、登录页面；
5. **前端业务模块阶段**：按优先级从高到低开发首页→进销存→用户→系统设置→流水报表；
6. **联调测试阶段**：前后端接口联调、功能测试、性能测试、安全测试。

### 6.2 依赖关系
- 后端业务模块依赖后端核心基础阶段；
- 前端业务模块依赖前端基础阶段；
- 联调测试阶段依赖前后端业务模块阶段。

### 6.3 执行优先级
- 优先级1（P0）：数据库阶段、后端核心基础阶段、前端基础阶段、进销存模块（商品档案、采购管理、销售管理、库存管理）、用户模块（员工管理、会员管理）、系统设置模块（基础参数设置、权限配置）、首页模块；
- 优先级2（P1）：流水报表模块、系统设置模块（接口配置、第三方短信接口配置、系统日志）、库存盘点管理模块；
- 优先级3（P2）：通用前端对接接口、系统设置模块（敏感操作二次验证设置）。

## 7. 代码生成规范
### 7.1 命名规范
- **类名**：大驼峰命名，必须包含核心业务领域关键词，如`BeautyPurchaseOrderController`、`BeautySalesOrderService`、`BeautyInventoryProductMapper`；
- **方法名**：小驼峰命名，必须清晰表达业务含义，如`saveBeautyPurchaseOrder`、`getBeautyInventoryProductList`；
- **变量名**：小驼峰命名，必须清晰表达变量含义，如`beautyPurchaseOrderNo`、`beautyMemberPhone`；
- **表名**：小写字母+下划线分隔，必须以`beauty_`开头，如`beauty_purchase_order`、`beauty_sales_order_detail`；
- **字段名**：小写字母+下划线分隔，必须清晰表达字段含义，如`beauty_purchase_order_no`、`beauty_member_phone`。

### 7.2 代码结构
- **后端**：严格遵循Controller/Service/Mapper三层架构，Service层分为接口和实现；
- **前端**：严格遵循API/Views/Components/Utils/Store/Router分层结构。

### 7.3 注释要求
- **类注释**：使用JavaDoc/VueDoc注释，包含类的功能描述、作者、创建时间；
- **方法注释**：使用JavaDoc/VueDoc注释，包含方法的功能描述、请求参数、响应格式、业务逻辑、异常处理；
- **复杂业务逻辑**：必须添加单行/多行注释，清晰表达逻辑步骤。

### 7.4 错误码定义
- 错误码前缀必须包含核心业务领域标识`BEAUTY`；
- 错误码格式为`BEAUTY_XXX`，其中XXX为三位数字；
- 错误码示例：
  - `BEAUTY_001`：参数校验失败；
  - `BEAUTY_002`：用户名或密码错误；
  - `BEAUTY_003`：商品库存不足；
  - `BEAUTY_004`：敏感操作需二次验证；
  - `BEAUTY_500`：系统内部错误。

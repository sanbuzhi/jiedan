# TASKS.md
## 1. 项目技术规格
### 技术栈
- **后端**：Java 17 + Spring Boot 3.2.5 + MyBatis Plus 3.5.5 + Swagger 3.2.0 + Logback + JWT + AES-256
- **管理端前端**：Vue 3.4.21 + Element Plus 2.6.3 + Vite 5.2.11 + ECharts 5.5.0 + xlsx 0.18.5 + jspdf-autotable 3.5.31
- **数据库**：MySQL 8.0.36（InnoDB引擎） + Redis 7.2.5（热点数据缓存）
- **服务器**：Nginx 1.24.0 + Spring Boot内置Tomcat

### 项目结构规范
#### 后端（Maven多模块）
```
beauty-shop-backend
├── beauty-common       # 公共模块（工具类、常量、异常、统一响应）
├── beauty-config       # 配置模块（数据源、Redis、JWT、Swagger）
├── beauty-entity       # 数据库实体类（含自动填充字段）
├── beauty-dto          # 请求/响应DTO（含校验规则）
├── beauty-vo           # 视图对象（含报表统计、下拉选项）
├── beauty-mapper       # MyBatis Plus Mapper（含自定义SQL）
├── beauty-service      # 业务逻辑层（含接口+实现）
├── beauty-controller   # 控制器层（RESTful API）
└── beauty-shop-admin   # 管理端启动模块
```

#### 管理端前端（Vue3 + Element Plus）
```
beauty-shop-admin-front
├── public              # 静态资源（favicon）
├── src
│   ├── api             # 接口调用（按业务模块分）
│   ├── assets          # 资源文件（logo、CSS）
│   ├── components      # 公共组件（扫码枪监听、小票预览、分页表格）
│   ├── router          # 路由配置（含权限守卫）
│   ├── store           # Pinia状态管理（用户信息、系统参数、购物车）
│   ├── utils           # 工具函数（加密、日期、Excel/PDF导出）
│   ├── views           # 业务页面（按模块分）
│   ├── App.vue         # 根组件
│   └── main.js         # 入口文件
├── vite.config.js
├── package.json
└── README.md
```

### 代码生成规则
- **依赖版本**：严格使用上述技术栈的指定版本，所有依赖从Maven Central/NPM官方源获取
- **数据校验**：使用Hibernate Validator实现DTO的参数校验（@NotNull、@Size、@Min、@Max等）
- **JWT配置**：
  - Token有效期默认7天，刷新Token有效期30天
  - 使用HS256加密算法，密钥长度≥64位
  - Token中包含员工ID、角色ID、权限列表
- **加密配置**：敏感数据（密码、手机号、API Secret）使用AES-256加密存储
- **自动填充**：使用MyBatis Plus自动填充创建时间、更新时间、创建人ID、更新人ID

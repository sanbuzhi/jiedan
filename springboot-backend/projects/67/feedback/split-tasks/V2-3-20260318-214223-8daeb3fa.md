# TASKS.md
## 1. 项目技术规格
### 技术栈
- **后端**：Java 17 + Spring Boot 3.2.5 + MyBatis Plus 3.5.5 + Swagger 3.2.0 + Logback + JWT + AES-256
- **管理端前端**：Vue 3.4.21 + Element Plus 2.6.3 + Vite 5.2.11 + ECharts 5.5.0 + xlsx 0.18.5 + jspdf-autotable 3.5.31
- **数据库**：MySQL 8.0.36（InnoDB引擎，utf8mb4字符集） + Redis 7.2.5（热点数据缓存）
- **服务器**：Nginx 1.24.0 + Spring Boot内置Tomcat 10.1.24

### 项目结构规范
#### 后端（Maven多模块）
```
beauty-shop-backend
├── beauty-common       # 公共模块（工具类、常量、统一异常、统一响应）
├── beauty-config       # 配置模块（数据源、Redis、JWT、Swagger、MyBatis Plus）
├── beauty-entity       # 数据库实体类（含自动填充字段、逻辑删除标识）
├── beauty-dto          # 请求/响应DTO（含Hibernate Validator校验规则）
├── beauty-vo           # 视图对象（含报表统计、下拉选项、预警详情）
├── beauty-mapper       # MyBatis Plus Mapper（含自定义SQL、分页插件配置）
├── beauty-service      # 业务逻辑层（含接口+实现，默认开启事务）
├── beauty-controller   # 控制器层（RESTful API，含权限注解、日志注解）
└── beauty-shop-admin   # 管理端启动模块（含启动类、全局配置）
```

#### 管理端前端（Vue3 + Element Plus）
```
beauty-shop-admin-front
├── public              # 静态资源（favicon.ico）
├── src
│   ├── api             # 接口调用（按业务模块分，封装统一请求拦截/响应拦截）
```

### 代码生成规则
- **依赖版本**：严格使用上述技术栈的指定版本，所有依赖从Maven Central/NPM官方源获取
- **数据校验**：使用Hibernate Validator实现DTO的参数校验（@NotNull、@Size、@Min、@Max、@DecimalMin、@Pattern等）
- **JWT配置**：
  - Token有效期默认7天，刷新Token有效期30天
  - 使用HS256加密算法，密钥长度≥64位
  - Token中包含员工ID、角色ID、权限列表
- **加密配置**：敏感数据（密码、手机号、API Secret）使用AES-256加密存储
- **自动填充**：使用MyBatis Plus自动填充创建时间、更新时间、创建人ID、更新人ID
- **逻辑删除**：所有业务表使用逻辑删除（deleted字段，TINYINT(1)，0未删除1已删除）

## 5. 业务逻辑规则
### 5.1 核心业务流程
#### 5.1.1 美妆SKU采购入库流程
1. 有采购权限的员工登录系统 → 进入「采购管理」→「创建采购单」
2. 填写供应商信息（正式/临时）→ 扫码/搜索添加美妆SKU明细 → 调整折扣率 → 自动计算小计/总金额
3. 选择支付方式 → 点击「保存并提交」生成状态为“待入库”的采购单 → 应付账款支付方式下自动增加供应商应付余额
4. 有入库权限的员工登录系统 → 进入「采购管理」→「待入库/部分入库采购单」
5. 扫码/输入采购单号检索 → 逐一核对美妆SKU明细 → 调整入库数量/库位 → 点击「确认入库」
6. 更新采购单状态 → 同步更新美妆SKU库存/总金额 → 生成入库记录单 → 全部入库完成提示是否结算应付账款

#### 5.1.2 美妆零售单创建流程
1. 有收银权限的员工登录系统 → 进入「销售管理」→「创建零售单」
2. 扫码/搜索添加会员（或跳过为散客）→ 自动加载会员等级折扣
3. 扫码/搜索添加美妆SKU明细 → 自动显示库存充足的库位 → 调整销售数量/单价（≥最低零售价）/折扣率 → 自动计算小计/总金额
4. 选择抹零/抹分规则 → 选择支付方式 → 点击「确认收款」
5. 生成销售单号 → 同步更新美妆SKU库存/总金额 → 同步更新会员信息（消费次数、累计金额、积分、等级进度）→ 同步扣减储值/积分（若使用）→ 打印小票

#### 5.1.3 美妆库存盘点流程
1. 有创建盘点单权限的员工登录系统 → 进入「库存管理」→「库存盘点管理」→「创建盘点单」
2. 选择盘点类型（全盘/类别盘/库位盘/单品盘）→ 选择盘点范围 → 点击「保存」
3. 生成状态为“待盘点”的盘点单 → 自动锁定盘点范围的美妆SKU库存 → 导出盘点初稿
4. 有实盘权限的员工登录系统 → 进入「库存盘点管理」→「实盘录入」
5. 扫码/输入盘点单号检索 → 逐一录入实盘数量/库位 → 差异备注必填（若实盘≠系统库存）→ 点击「提交审核」
6. 生成状态为“待审核”的盘点单 → 有审核权限的员工审核 → 审核通过则调整库存/解锁/生成盘盈盘亏记录，审核驳回则修改实盘明细重新提交

### 5.2 数据校验规则
#### 5.2.1 美妆SKU相关校验
- 商品条码：唯一，支持EAN-13/UPC-A/Code-128格式
- 生产日期：≤当前系统时间
- 保质期：≥1个月（若填写）
- 最低零售价：≥成本价
- 建议零售价：≥最低零售价
- 会员等级折扣：≤普通会员≤银卡会员≤金卡会员（若覆盖通用折扣）

#### 5.2.2 采购相关校验
- 采购数量：≥1
- 入库数量：≤采购单剩余待入库数量
- 临时供应商：需填写名称+联系方式

#### 5.2.3 销售相关校验
- 销售数量：≥1
- 销售单价：≥最低零售价
- 库存数量：销售数量≤（库存数量-锁定数量）
- 积分抵扣：需关联会员，可用积分≥抵扣积分
- 储值消费：需关联会员，储值余额≥储值消费金额
- 销售日期：≥对应美妆SKU的最后入库日期

### 5.3 状态流转规则
#### 5.3.1 采购单状态流转
- 草稿 → 待入库：点击「保存并提交」
- 待入库 → 部分入库：部分美妆SKU明细完成入库
- 部分入库 → 已完成：所有美妆SKU明细完成入库
- 草稿/部分入库/已完成：不可删除，仅草稿可编辑

#### 5.3.2 销售单状态流转
- 草稿 → 待发货（线上）/已完成（线下零售）：点击「确认收款」
- 待发货 → 已发货：点击「确认发货」
- 已发货/已完成 → 已部分退货：部分美妆SKU明细完成退货
- 已部分退货 → 已全部退货：所有美妆SKU明细完成退货
- 待发货/已发货 → 已取消：点击「确认取消」

### 5.4 异常处理规则
- 库存不足：零售/线上订单创建时抛出BEAUTY_SALES_001异常，提示“美妆SKU[xxx]库存不足”
- 库存锁定：盘点锁定期间操作库存抛出BEAUTY_INVENTORY_001异常，提示“美妆SKU[xxx]正在盘点中，请稍后再试”
- 跨权限操作：抛出BEAUTY_AUTH_001异常，记录错误日志
- 签名验证失败：通用前端接口抛出BEAUTY_FRONT_001异常
- 短信发送失败：抛出BEAUTY_SMS_001异常，记录错误日志，不影响核心业务流程

## 7. 代码生成规范
### 7.1 命名规范
#### 7.1.1 后端命名
- **类名**：大驼峰，必须包含“Beauty”前缀，如BeautyProductController、BeautyProductService、BeautyProductServiceImpl、BeautyProduct
- **方法名**：小驼峰，必须贴合美妆领域，如saveBeautyProduct、listBeautyPurchaseOrder、adjustBeautyMemberPoints
- **变量名**：小驼峰，必须贴合美妆领域，如beautyProductId、beautyPurchaseNo、beautyMemberDiscount
- **常量名**：全大写下划线分隔，必须包含“BEAUTY”前缀，如BEAUTY_PURCHASE_NO_PREFIX、BEAUTY_WARNING_TYPE_SHORTAGE
- **表名**：全小写下划线分隔，必须包含“beauty_”前缀，如beauty_product、beauty_purchase_order
- **字段名**：全小写下划线分隔，必须贴合美妆领域，如beauty_product_no、beauty_member_discount

#### 7.1.2 前端命名
- **文件/文件夹名**：全小写下划线分隔，必须贴合美妆领域，如beauty_product_list.vue、beauty_purchase
- **组件名**：大驼峰，必须包含“Beauty”前缀，如BeautyProductTable、BeautyScanGunListener
- **变量名**：小驼峰，必须贴合美妆领域，如beautyProductList、beautyPurchaseNo
- **接口方法名**：小驼峰，必须包含“get/post/put/delete”前缀，贴合美妆领域，如getBeautyProductList、postBeautyPurchaseOrder

### 7.2 代码结构
#### 7.2.1 后端分层
- **Controller层**：负责接收请求、参数校验、权限校验、调用Service层、返回统一响应
- **Service层**：负责核心业务逻辑、事务管理、调用Mapper层、抛出统一异常
- **Mapper层**：负责数据库操作，使用MyBatis Plus自动生成的方法+自定义SQL

#### 7.2.2 前端分层
- **API层**：负责封装统一请求拦截/响应拦截、调用后端接口
- **Store层**：负责管理全局状态（用户信息、系统参数、购物车）
- **View层**：负责页面展示、用户交互、调用API层/Store层
- **Component层**：负责封装公共组件，复用代码

### 7.3 注释要求
#### 7.3.1 后端注释
- **类注释**：使用JavaDoc，包含类描述、作者、创建时间
- **方法注释**：使用JavaDoc，包含方法描述、参数说明、返回值说明、异常说明
- **复杂逻辑注释**：使用单行注释，说明逻辑的目的、步骤
- **数据库字段注释**：在实体类中使用@TableField注解的comment属性，在数据库表中添加COMMENT

#### 7.3.2 前端注释
- **组件注释**：使用/** */注释，包含组件描述、props说明、events说明
- **函数注释**：使用/** */注释，包含函数描述、参数说明、返回值说明
- **复杂逻辑注释**：使用//注释，说明逻辑的目的、步骤

### 7.4 错误码定义
错误码格式：BEAUTY_XXXXX（XX为模块编号，XXX为顺序号）
- **BEAUTY_00001**：系统错误
- **BEAUTY_10001**：认证错误（账号/密码错误、Token过期、Token无效）
- **BEAUTY_10002**：权限错误（跨权限操作、无权限访问）
- **BEAUTY_20001**：商品错误（商品不存在、库存不足、库存锁定）
- **BEAUTY_20002**：采购错误（采购单不存在、入库数量超限）
- **BEAUTY_20003**：销售错误（销售单不存在、销售单价过低、积分/储值不足）
- **BEAUTY_20004**：库存错误（库存调拨数量超限、盘点单不存在）
- **BEAUTY_30001**：用户错误（员工/会员不存在、手机号重复）
- **BEAUTY_40001**：报表错误（报表数据生成失败）
- **BEAUTY_50001**：系统设置错误（API Key/Secret生成失败）
- **BEAUTY_60001**：通用前端错误（签名验证失败、订单数据不完整）
- **BEAUTY_70001**：短信错误（短信发送失败）

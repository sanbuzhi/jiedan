# 美妆小店轻量级后端管理系统V1.0 需求规格说明书
---

## 需求概述
### 项目名称
美妆小店轻量级后端管理系统V1.0
### 项目类型
单店专属B2C垂直电商后端管理系统（网页版）
### 项目背景
店主当前通过「零散Excel表格记录+微信人工收单+收银小票手工录入」的方式管理门店，存在以下核心痛点：
1. 采购入库、销售出库操作无系统流程，库存盘点耗时1-2小时且准确率不足90%；
2. 销售、会员、采购、支出等流水分散，无法快速生成日报/周报/月报/单品分析等财务运营报表；
3. 员工操作无权限划分，查账、修改库存等敏感操作风险高；
4. 对接无线上接口，用户微信下单后需人工同步库存和订单，效率低易漏单。
### 项目目标
1. 实现进销存全流程数字化，库存盘点耗时≤30分钟、准确率≥99%；
2. 支持一键生成多维度流水报表，单报表导出时间≤5秒；
3. 建立三级员工权限体系，敏感操作需二次验证；
4. 对接通用美妆店前端零售小程序，实现订单、库存、会员数据自动同步。
### 项目范围
#### 包含范围
1. 进销存管理模块：采购管理、销售管理、库存管理（含库存盘点）；
2. 用户管理模块：员工管理、会员管理；
3. 流水报表模块：财务流水类报表、业务运营类报表；
4. 系统设置模块：基础参数设置、权限配置、接口配置、系统日志；
5. 通用前端对接接口：用户下单、库存查询、会员信息同步等。
#### 不包含范围
1. 前端零售小程序/网页的开发；
2. 线下POS机的硬件对接；
3. 多门店管理功能；
4. 直播带货、分销等营销管理功能；
5. 第三方支付平台的直接结算（仅需记录支付流水）。

## 1. 功能需求
### 1.1 进销存管理模块
#### 1.1.1 采购管理
##### 子功能1：采购单创建
- **输入参数**：
  1. 供应商信息（下拉选择已录入供应商，或临时填写供应商名称+联系方式，临时供应商需提示“是否保存至供应商库”）；
  2. 采购商品明细（支持扫码枪/手动输入商品条码/名称检索，单条明细必填：商品SKU、采购数量、采购单价、折扣率（默认100%，可手动调整）、小计金额（系统自动计算：采购数量×采购单价×折扣率））；
  3. 采购总金额（系统自动汇总所有商品明细小计金额）；
  4. 采购日期（默认当前系统时间，可手动调整）；
  5. 采购备注（可选，最多200字）；
  6. 支付方式（下拉选择：现金、微信、支付宝、银行转账、应付账款）。
- **输出结果**：
  1. 系统自动生成采购单号（格式为CG+8位年月日+4位流水号，如CG202405200001）；
  2. 生成状态为“待入库”的采购单，支持导出为Excel/PDF；
  3. 应付账款支付方式下，自动增加该供应商的应付账款余额。
- **业务流程**：
  1. 有采购权限的员工登录系统；
  2. 进入「采购管理」→「创建采购单」页面；
  3. 填写所有必填参数；
  4. 点击「保存并提交」生成采购单；
  5. 点击「保存草稿」可暂存未完成的采购单，后续可在「采购草稿」中修改提交。

##### 子功能2：采购单入库
- **输入参数**：
  1. 采购单号（输入/扫码枪扫描采购单条码检索）；
  2. 商品入库明细（支持扫码枪/手动输入商品条码/名称逐一核对，单条明细必填：入库数量（默认采购单剩余待入库数量，可调整≤剩余待入库数量）、入库库位（下拉选择已录入库位，默认“主仓库”）、入库备注（可选，最多100字））；
  3. 入库日期（默认当前系统时间，可手动调整）；
  4. 入库员信息（自动获取当前登录员工信息，不可修改）。
- **输出结果**：
  1. 更新采购单状态：若所有商品明细入库完成则为“已完成”，若部分完成则为“部分入库”；
  2. 同步更新对应商品SKU的库存数量（入库库位+入库数量）、库存总金额（系统自动计算：入库数量×采购单价×折扣率）；
  3. 生成入库记录单，支持导出为Excel/PDF；
  4. 若采购单全部入库完成，且支付方式为应付账款，自动提示“是否立即结算该供应商应付账款”。
- **业务流程**：
  1. 有入库权限的员工登录系统；
  2. 进入「采购管理」→「待入库/部分入库采购单」列表；
  3. 选择需要入库的采购单，点击「入库」按钮；
  4. 逐一核对并填写入库明细；
  5. 点击「确认入库」完成操作。

##### 子功能3：采购单查询/编辑/删除
- **查询条件**：采购单号、供应商名称/联系方式、采购日期范围、采购单状态、支付方式、创建人。
- **编辑权限**：仅状态为“草稿”的采购单可由创建人或有采购管理权限的管理员编辑。
- **删除权限**：仅状态为“草稿”的采购单可由创建人或有采购管理权限的管理员删除。

##### 子功能4：供应商管理
- **输入参数**：供应商名称（必填）、联系方式（必填，手机号/座机号二选一）、地址（可选）、供应商品类别（可选，多选已录入的商品类别）、合作折扣率（可选，默认100%，创建采购单时可覆盖）。
- **输出结果**：生成供应商档案，支持扫码枪扫描供应商档案条码检索。
- **功能项**：供应商档案创建、查询、编辑、删除（未关联任何有效采购单的供应商可删除）。

#### 1.1.2 销售管理
##### 子功能1：零售单创建（线下场景）
- **输入参数**：
  1. 会员信息（可选，扫码枪/手动输入会员手机号/卡号检索，或直接跳过为散客）；
  2. 销售商品明细（支持扫码枪/手动输入商品条码/名称检索，单条明细必填：商品SKU、销售数量、销售单价（默认商品档案中的“建议零售价”，可手动调整≥“最低零售价”）、折扣率（默认100%，可手动调整，若关联会员则优先使用会员等级折扣）、小计金额（系统自动计算：销售数量×销售单价×折扣率）、库存库位（自动显示商品库存充足的库位，可手动调整））；
  3. 销售总金额（系统自动汇总所有商品明细小计金额，支持抹零/抹分，抹零/抹分后的金额需大于等于0）；
  4. 销售日期（默认当前系统时间，可手动调整，但需晚于商品入库日期）；
  5. 销售备注（可选，最多100字）；
  6. 支付方式（下拉选择：现金、微信、支付宝、银行转账、储值卡、积分抵扣（需关联会员））；
  7. 收银员信息（自动获取当前登录员工信息，不可修改）。
- **输出结果**：
  1. 系统自动生成销售单号（格式为XS+8位年月日+4位流水号，如XS202405200001）；
  2. 同步更新对应商品SKU的库存数量（销售库位-销售数量）、库存总金额；
  3. 若关联会员：同步增加会员消费次数、累计消费金额、可用积分（消费金额按比例换算，比例在系统设置中配置）、等级进度；
  4. 若使用储值卡：同步扣减会员储值余额；
  5. 若使用积分抵扣：同步扣减会员可用积分；
  6. 生成零售单，支持导出为Excel/PDF、打印小票（小票模板在系统设置中配置）。
- **业务流程**：
  1. 有收银权限的员工登录系统；
  2. 进入「销售管理」→「创建零售单」页面；
  3. 填写所有必填参数；
  4. 点击「确认收款」完成操作；
  5. 若支付金额不足，自动提示差额并返回修改。

##### 子功能2：线上订单同步（对接通用前端场景）
- **输入参数**：前端小程序传入的订单数据（订单号、会员信息、商品明细、支付方式、支付金额、下单时间、收货地址、收货人联系方式等）。
- **输出结果**：
  1. 系统自动生成状态为“待发货”的销售单；
  2. 同步更新对应商品SKU的库存数量（自动分配库存充足的主仓库，若主仓库库存不足则锁定其他库位库存并提示管理员/发货员）；
  3. 同步更新关联会员的相关数据（同零售单创建）；
  4. 生成待发货记录，进入「待发货订单」列表。
- **业务流程**：
  1. 通用前端小程序调用系统接口提交订单；
  2. 系统接口验证订单数据的完整性、商品库存的可用性；
  3. 验证通过后生成销售单并更新数据；
  4. 验证失败后返回错误信息给前端。

##### 子功能3：线上订单发货/取消
- **发货输入参数**：
  1. 销售单号（输入/扫码枪扫描销售单条码检索）；
  2. 发货商品明细（自动显示订单商品，可核对）；
  3. 物流公司（下拉选择已录入的物流公司，或临时填写）；
  4. 物流单号（必填）；
  5. 发货日期（默认当前系统时间，可手动调整）；
  6. 发货员信息（自动获取当前登录员工信息，不可修改）。
- **发货输出结果**：
  1. 更新销售单状态为“已发货”；
  2. 调用通用前端接口同步发货状态和物流信息；
  3. 生成发货记录单，支持导出为Excel/PDF。
- **取消输入参数**：
  1. 销售单号（输入/扫码枪扫描销售单条码检索）；
  2. 取消原因（必填，下拉选择：用户申请、库存不足、其他，需补充其他原因的描述）；
  3. 审核人信息（自动获取当前登录有取消权限的员工信息，不可修改）。
- **取消输出结果**：
  1. 更新销售单状态为“已取消”；
  2. 同步释放锁定/扣减的商品库存；
  3. 同步退还关联会员的储值余额/可用积分；
  4. 调用通用前端接口同步取消状态。

##### 子功能4：销售单查询/退货
- **查询条件**：销售单号、会员信息、商品SKU/名称、销售日期范围、销售单状态、支付方式、收银员/发货员。
- **退货输入参数**：
  1. 销售单号（输入/扫码枪扫描销售单条码检索）；
  2. 退货商品明细（支持扫码枪/手动输入商品条码/名称检索，单条明细必填：商品SKU、退货数量（默认≤原销售数量）、退货库位（默认“主仓库”，下拉选择已录入库位）、退货原因（必填，下拉选择：质量问题、用户后悔、其他，需补充其他原因的描述）、退款金额（系统自动计算：原商品明细小计金额×退货数量/原销售数量-积分抵扣金额×退货数量/原销售数量，可手动调整≥0））；
  3. 退货总金额（系统自动汇总所有商品明细退款金额）；
  4. 退款方式（默认原支付方式，可调整：现金、微信、支付宝、银行转账、储值卡）；
  5. 退货日期（默认当前系统时间，可手动调整，但需晚于销售日期）；
  6. 退货审核人信息（自动获取当前登录有退货权限的员工信息，不可修改）。
- **退货输出结果**：
  1. 系统自动生成退货单号（格式为TH+8位年月日+4位流水号，如TH202405200001）；
  2. 更新原销售单状态为“已部分退货”或“已全部退货”；
  3. 同步更新对应商品SKU的库存数量（退货库位+退货数量）、库存总金额；
  4. 同步扣减关联会员的消费次数、累计消费金额、等级进度；
  5. 若使用原积分抵扣，同步退还会员可用积分；
  6. 若退款方式为储值卡，同步增加会员储值余额；
  7. 生成退货记录单，支持导出为Excel/PDF。

#### 1.1.3 库存管理
##### 子功能1：商品档案管理
- **输入参数**：
  1. 商品基本信息：商品条码（必填，支持扫码枪扫描，支持批量生成条码规则：商品类别编码+流水号）、商品名称（必填）、商品类别（必填，下拉选择已录入的商品类别）、品牌（可选）、规格（可选，如50ml、片装）、生产日期（可选，需早于或等于当前系统时间）、保质期（可选，如36个月）、过期预警天数（可选，默认商品类别设置的预警天数，可覆盖）；
  2. 商品价格信息：成本价（必填）、最低零售价（必填，≥成本价）、建议零售价（必填，≥最低零售价）、会员等级折扣（可选，默认继承会员等级的通用折扣，可覆盖）；
  3. 商品库存信息：安全库存（可选，默认商品类别设置的安全库存，可覆盖）、初始库存（可选，仅在首次创建商品档案时填写，系统自动生成初始入库记录）、初始库存库位（默认“主仓库”，下拉选择已录入库位）；
  4. 商品图片（可选，支持上传JPG/PNG格式，单张图片≤2MB，最多5张）。
- **输出结果**：生成商品档案，支持扫码枪扫描商品条码/档案条码检索。
- **功能项**：商品档案创建、查询、编辑（成本价、最低零售价、建议零售价、会员等级折扣的修改需记录系统日志）、删除（未关联任何有效采购单、销售单、库存记录的商品可删除）、批量导入（支持Excel批量导入商品档案，需提供模板下载）。

##### 子功能2：商品类别管理
- **输入参数**：商品类别名称（必填）、父类别（可选，默认“美妆全品类”）、通用过期预警天数（可选，默认30天）、通用安全库存（可选，默认10件）。
- **输出结果**：生成商品类别树状结构。
- **功能项**：商品类别创建、查询、编辑、删除（未关联任何商品的子类别可删除，“美妆全品类”默认不可删除）。

##### 子功能3：库位管理
- **输入参数**：库位名称（必填，如主仓库-口红区-第一层）、库位容量（可选，单位为件）、库位状态（必填，下拉选择：正常、停用）。
- **输出结果**：生成库位档案。
- **功能项**：库位档案创建、查询、编辑、删除（未关联任何商品库存的库位可删除）。

##### 子功能4：库存预警
- **预警类型**：
  1. 库存不足预警：商品SKU库存总数量≤安全库存；
  2. 库存积压预警：商品SKU库存总数量≥库位总容量的90%（可在系统设置中调整比例）；
  3. 商品过期预警：商品SKU的库存生产日期+保质期≤当前系统时间+过期预警天数。
- **输出结果**：
  1. 系统首页顶部显示预警数量统计；
  2. 进入「库存管理」→「库存预警」页面可查看预警详情；
  3. 库存不足预警支持直接点击「创建采购单」跳转至采购单创建页面，并自动填充预警商品明细；
  4. 商品过期预警支持导出为Excel/PDF。

##### 子功能5：库存调拨
- **输入参数**：
  1. 调拨商品明细（支持扫码枪/手动输入商品条码/名称检索，单条明细必填：商品SKU、调拨数量（默认≤原库位剩余库存）、原库位（必填，下拉选择已录入的正常库位）、目标库位（必填，下拉选择已录入的正常库位，且≠原库位））；
  2. 调拨日期（默认当前系统时间，可手动调整）；
  3. 调拨备注（可选，最多100字）；
  4. 调拨审核人信息（自动获取当前登录有调拨权限的员工信息，不可修改）。
- **输出结果**：
  1. 系统自动生成调拨单号（格式为DB+8位年月日+4位流水号，如DB202405200001）；
  2. 同步更新对应商品SKU的原库位库存数量（-调拨数量）、目标库位库存数量（+调拨数量）；
  3. 生成调拨记录单，支持导出为Excel/PDF。

#### 1.1.4 库存盘点管理
##### 子功能1：盘点单创建
- **输入参数**：
  1. 盘点单号（系统自动生成，格式为PD+8位年月日+4位流水号，如PD202405200001）；
  2. 盘点类型（必填，下拉选择：全盘、类别盘、库位盘、单品盘）；
  3. 盘点范围（必填，根据盘点类型选择：
     - 全盘：自动覆盖所有正常库位的所有商品；
     - 类别盘：多选已录入的商品类别；
     - 库位盘：多选已录入的正常库位；
     - 单品盘：扫码枪/手动输入商品条码/名称检索并多选）；
  4. 盘点日期（默认当前系统时间，可手动调整）；
  5. 盘点备注（可选，最多100字）；
  6. 盘点人信息（自动获取当前登录有创建盘点单权限的员工信息，不可修改）。
- **输出结果**：
  1. 生成状态为“待盘点”的盘点单，支持导出为Excel/PDF（仅包含盘点范围的商品基本信息和系统库存数量，无实盘数量）；
  2. 系统自动锁定盘点范围的商品库存（锁定期间不允许采购入库、销售出库、退货入库、库存调拨操作，直到盘点单状态为“已完成”或“已取消”）。

##### 子功能2：盘点单实盘录入
- **输入参数**：
  1. 盘点单号（输入/扫码枪扫描盘点单条码检索）；
  2. 实盘明细（支持扫码枪/手动输入商品条码/名称逐一录入，单条明细必填：商品SKU、实盘数量（默认为0，可调整）、实盘库位（默认系统库存所在库位，可调整≤盘点范围的库位）、差异备注（可选，若实盘数量≠系统库存数量则为必填））。
- **输出结果**：
  1. 系统自动计算实盘数量与系统库存数量的差异（差异数量=实盘数量-系统库存数量）、差异金额（差异金额=差异数量×成本价）；
  2. 更新盘点单状态：若所有盘点范围的商品都已录入实盘数量则为“待审核”，若部分录入则为“实盘中”。

##### 子功能3：盘点单审核
- **输入参数**：
  1. 盘点单号（输入/扫码枪扫描盘点单条码检索）；
  2. 审核结果（必填，下拉选择：通过、驳回）；
  3. 审核备注（可选，若驳回则为必填）；
  4. 审核人信息（自动获取当前登录有盘点审核权限的员工信息，不可修改）。
- **输出结果**：
  1. 若审核通过：
     - 更新盘点单状态为“已完成”；
     - 同步更新对应商品SKU的库存数量（差异调整）、库存总金额；
     - 生成盘盈盘亏记录单，支持导出为Excel/PDF；
     - 自动解锁盘点范围的商品库存。
  2. 若审核驳回：
     - 更新盘点单状态为“已驳回”；
     - 支持创建盘点单的员工修改实盘明细后重新提交审核；
     - 盘点范围的商品库存仍处于锁定状态，直到重新审核通过或盘点单取消。

##### 子功能4：盘点单查询/取消
- **查询条件**：盘点单号、盘点类型、盘点日期范围、盘点单状态、盘点人/审核人。
- **取消输入参数**：
  1. 盘点单号（输入/扫码枪扫描盘点单条码检索）；
  2. 取消原因（必填，下拉选择：盘点范围错误、实盘人员请假、其他，需补充其他原因的描述）；
  3. 取消审核人信息（自动获取当前登录有取消权限的员工信息，不可修改）。
- **取消输出结果**：
  1. 更新盘点单状态为“已取消”；
  2. 自动解锁盘点范围的商品库存。

#### 1.4.2 敏感操作二次验证设置
- **功能项**：
  1. 敏感操作列表配置（必填，默认勾选：修改商品成本价/最低零售价/建议零售价、修改员工角色/删除员工/重置员工密码、修改会员等级/删除会员/调整会员初始储值余额/初始可用积分/大额储值充值（≥X元，X默认500，可手动调整）、删除/修改已审核通过的采购单/销售单/盘点单、导出系统日志），可手动取消勾选（但删除已审核通过的单据默认不可取消）；
  2. 二次验证方式配置（必填，默认勾选“短信验证码验证”，可额外勾选“指纹验证（仅支持管理员指纹库提前录入的本地设备）”，必须保留至少一种）；
  3. 短信验证码有效期（默认5分钟，可手动调整1-30分钟）；
  4. 短信验证码错误次数限制（默认3次，可手动调整1-10次，达到限制后锁定二次验证功能1小时）。

#### 1.4.3 接口配置
- **通用前端对接接口**：
  1. 对接方式：RESTful API，请求/响应格式统一为UTF-8编码的JSON；
  2. 鉴权方式：
     - 首次对接：管理员在系统设置中生成唯一的API Key和API Secret，前端存储API Key；
     - 日常请求：前端使用API Key+当前时间戳（毫秒级）+32位随机数生成SHA256签名，系统验证签名的有效性和时间戳的合理性（默认允许误差≤5分钟，可手动调整1-30分钟）；
     - 特殊接口（如会员登录、储值消费）：前端需携带用户JWT Token，Token有效期默认7天，可手动调整1-30天，过期自动刷新；
  3. 核心功能接口概述：
     - 商品类：商品列表查询、商品详情查询、商品库存查询；
     - 会员类：会员注册/登录、会员信息查询/修改、会员储值余额/可用积分查询；
     - 订单类：线上订单创建、线上订单支付状态同步、线上订单状态查询、线上订单取消申请；
  4. 功能项：API Key/Secret生成、修改、删除（仅支持删除未对接的API Key）、签名验证规则调整、模拟测试接口功能。
- **第三方短信接口**：
  1. 对接方式：支持阿里云短信、腾讯云短信、百度云短信等主流第三方服务商的标准接口；
  2. 参数配置（以阿里云为例，其他服务商需适配）：
     - AccessKey ID/AccessKey Secret；
     - 已审核通过的短信签名；
     - 初始登录密码、会员注册、储值充值、线上订单发货、线上订单取消、敏感操作验证对应的模板ID；
  3. 功能项：参数配置、单条测试短信发送、短信发送记录查询（含发送时间、结果、错误信息）。

#### 1.4.4 系统日志
- **日志类型**：
  1. 登录/登出日志：记录所有员工的登录时间、IP、设备、结果（成功/失败）、登出时间；
  2. 敏感操作日志：记录所有触发二次验证的操作的时间、IP、设备、操作人、操作内容、验证方式、验证结果；
  3. 普通操作日志：记录所有未触发二次验证的操作的时间、IP、设备、操作人、操作内容；
  4. 系统错误日志：记录系统运行时的错误时间、模块、信息、堆栈；
- **查询条件**：日志类型、时间范围、操作人、操作IP、结果；
- **保留期限**：
  1. 登录/登出、普通操作：默认1年，可调整3个月-3年；
  2. 敏感操作：默认3年，可调整1年-永久；
  3. 系统错误：默认6个月，可调整1个月-1年；
- **功能项**：日志查询、Excel/PDF导出（敏感操作不含完整错误堆栈）、系统错误自动上报（可选，默认关闭，可配置上报至指定邮箱）。

## 2. 非功能需求
### 2.1 性能需求
1. **响应时间**：
   - 系统首页加载≤2秒；
   - 普通查询（如商品档案、最近30天销售明细）≤1秒；
   - 复杂查询（如最近1年销售汇总、盘点明细导出前）≤3秒；
   - 数据导出（≤10万条）≤5秒；
   - 扫码枪输入识别≤0.2秒。
2. **并发量**：
   - 支持最多10个员工同时在线操作；
   - 支持最多100个通用前端用户同时调用接口下单/查询。
3. **数据存储能力**：
   - 支持最多100万条商品档案、1000万条流水记录、100万条会员档案（满足单店3-5年需求）。

### 2.2 安全需求
1. **身份认证**：
   - 员工必须用账号+密码登录，首次登录强制修改初始密码；
   - 密码强度：≥8位，含大小写字母、数字、特殊字符；
   - 密码有效期：默认90天，到期前7天自动提醒；
   - 支持开启“禁止重复登录”。
2. **数据安全**：
   - 敏感数据（密码、手机号、API Secret）用AES-256加密存储；
   - 数据库定期备份：每日凌晨2点全量、每周日凌晨3点增量，本地全量保留30天、增量7天，可选云端备份保留90天；
   - 仅管理员可触发二次验证后手动备份/恢复数据库。
3. **操作安全**：
   - 敏感操作必须记录日志；
   - 采购单/销售单草稿自动保存；
   - 系统自动拦截跨权限操作并记录错误。
4. **网络安全**：
   - 部署后必须使用HTTPS协议；
   - 支持开启“IP白名单”。

### 2.3 可用性需求
1. **系统全年可用性≥99.5%**；
2. **界面友好性**：简洁B端风格、操作流程清晰、支持F5/Ctrl+S等快捷键、所有元素有明确提示。

### 2.4 兼容性需求
1. **管理端浏览器**：Chrome 80+、Firefox 75+、Edge 80+、Safari 13+；
2. **服务器**：CentOS 7+、Ubuntu 18.04+ Linux系统。

## 3. 用户角色
### 3.1 店长
- **权限描述**：系统所有模块的所有操作权限，含数据库手动备份/恢复、采购单/销售单/盘点单审核。
- **业务场景**：每日查看预警和统计、每周调整采购计划、每月核对账目、管理员工/配置参数。

### 3.2 收银员
- **权限描述**：零售单创建、销售单查询、线上订单状态查询、会员档案创建/查询/修改、小额储值充值（<500元）、会员积分增加（如活动赠送）。
- **业务场景**：每日为散客/会员收银、打印小票、注册会员、赠送积分。

### 3.3 仓库管理员
- **权限描述**：采购单创建/入库/查询、供应商档案管理、线上订单发货、商品/类别/库位档案管理、库存预警查询、库存调拨、盘点单创建/实盘录入/查询/取消。
- **业务场景**：根据预警创建采购单、商品到货入库、处理线上发货、每月/季度盘点、管理商品库位。

### 3.4 财务员
- **权限描述**：所有采购/销售/退货/供应商账款查询、会员储值/积分流水查询、所有报表查询/导出。
- **业务场景**：每日核对当日流水、每周核对供应商账款、每月生成财务报表给店长。

## 4. 技术约束
### 4.1 技术栈
#### 4.1.1 后端
- **推荐**：Java 17 + Spring Boot 3.x + MyBatis Plus 3.x + Swagger 3.x；
- **备选**：Python 3.10 + Django 4.x + DRF 3.x + DRF-YASG。

- **必须集成**：通用前端RESTful API、主流第三方短信接口；

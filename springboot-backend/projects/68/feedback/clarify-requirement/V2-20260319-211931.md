# 童趣衣橱进销存POS管理系统
## 需求概述
### 项目名称
童趣衣橱单店Web端零售进销存POS管理系统
### 项目类型
单店独立部署的Web端垂直零售管理系统
### 项目背景
童趣衣橱是一家面向3-12岁儿童的线下实体服装店，目前采用手工记账+Excel辅助的方式管理业务：
1. 进销存数据录入错漏率高（出入库手工登记/收银/盘点三者数据难以实时同步）
2. 日报/周报/月报需手动整理Excel，耗时久且维度单一
3. 现有POS收银依赖第三方通用设备，与服装品类特性（如多尺码多颜色的SKU绑定、会员生日折扣提示）不匹配
4. 无小票打印自定义功能，无法突出店铺品牌信息
### 项目目标
1. 建立**SKU级**的全链路进销存体系，出入库/销售/盘点数据实时联动，将手工数据处理错漏率降至1%以下
2. 实现**多维度实时流水报表**生成（可按日期/品类/尺码/颜色/收银员/会员/支付方式等筛选），单份复杂报表生成时间≤3秒
3. 开发适配服装单店的**POS扫码收银台**，支持童装品类特性操作，单客结账效率≤30秒（不含特殊优惠申请）
4. 提供**可自定义模板的小票打印功能**，支持店铺Logo、促销信息、会员码的嵌入
### 项目范围
#### 包含范围
- 基础设置（用户角色/权限、SKU字典、会员体系、店铺信息、支付方式配置、小票模板）
- 进销存管理（采购入库、销售退货入库、商品调拨（单店内部区域/货架调拨）、库存盘点、库存预警）
- POS收银管理（扫码/手动输入SKU、童装多属性快速选品、会员识别/生日折扣/积分抵现、支付全流程、小票打印/补打）
- 流水报表管理（销售流水报表、采购流水报表、库存流水报表、会员消费报表、收银员业绩报表、利润分析报表）
#### 不包含范围
- 线上电商平台对接（如淘宝/抖音/微店）
- 多门店连锁管理（如总部权限、跨店调拨、多店合并报表）
- 供应商管理全流程（仅支持供应商名称/联系方式记录）
- 财务税务申报对接

## 1. 功能需求
### 1.1 基础设置模块
#### 1.1.1 用户角色与权限管理
- **功能点1：角色创建/编辑/删除**
  - 支持预设“超级管理员”“店长”“收银员”“仓库管理员”4种角色
  - 超级管理员可创建自定义角色，配置对应功能模块的“查看/新增/编辑/删除/审核”权限（预设角色权限不可修改）
  - 删除角色前需先将该角色下的所有用户转移至其他角色
- **功能点2：用户管理**
  - 超级管理员/店长可创建/编辑/删除用户，设置用户名、密码、绑定角色、手机号、入职日期等信息
  - 支持用户密码重置（仅超级管理员/店长可操作）
  - 所有操作需记录操作日志（操作人、操作时间、操作内容）

#### 1.1.2 SKU字典管理
- **功能点1：品类管理**
  - 支持创建/编辑/删除童装品类（如“上衣”“裤子”“外套”“配饰”“鞋类”），支持设置品类排序
- **功能点2：属性模板管理**
  - 针对每个品类支持创建/编辑/删除属性模板（如“上衣”属性模板默认含“尺码（如110/120/130）”“颜色（如红色/蓝色/粉色）”“材质（如纯棉/牛仔/羽绒）”）
  - 属性支持“单选”“多选”“文本输入”三种类型，尺码/颜色属性需支持批量预设
- **功能点3：SPU/SKU创建**
  - SPU（标准化产品单元）创建：录入SPU名称、所属品类、绑定属性模板、主图、吊牌价、备注等信息
  - SKU（库存保有单元）自动生成：绑定属性模板后，系统自动根据属性组合生成所有SKU，支持手动删除不需要的属性组合
  - 每个SKU需唯一标识（可手动输入/自动生成规则：品类编码+SPU编码+属性编码组合），支持上传SKU实物图
  - 库存盘点前需先创建所有在售SKU

#### 1.1.3 会员体系管理
- **功能点1：会员等级管理**
  - 支持创建/编辑/删除会员等级（如“普通会员”“银卡会员”“金卡会员”），设置等级名称、所需累计消费金额、折扣比例（仅针对吊牌价有效）、积分倍数（普通会员1倍，银卡1.2倍，金卡1.5倍等）
- **功能点2：会员管理**
  - 支持POS端/后台手动注册会员：录入姓名、手机号（必填，唯一标识）、生日、性别、家庭住址等信息
  - 支持会员信息编辑/删除/禁用（禁用后无法享受会员权益）
  - 支持会员积分查看/调整（仅超级管理员/店长可操作调整，调整需记录原因）

#### 1.1.4 店铺与打印配置
- **功能点1：店铺信息配置**
  - 录入店铺名称、地址、联系电话、营业时间、店铺Logo等信息，Logo将用于小票打印
- **功能点2：支付方式配置**
  - 支持创建/编辑/删除支付方式：名称、标识（如现金/CASH、支付宝/ALIPAY、微信支付/WECHAT、会员卡余额/MEMBER_CARD）、是否启用、是否支持找零（仅现金支持）
  - 会员卡余额支付需关联会员账户，余额不足时需提示切换支付方式
- **功能点3：小票模板配置**
  - 支持可视化编辑POS收银小票模板：可拖拽添加/删除模块（店铺Logo、店铺信息、交易时间、交易流水号、收银员、商品明细、优惠明细、支付明细、应收/实收/找零、促销文字、会员码、售后提示）
  - 支持设置小票字体大小、行间距、对齐方式
  - 支持保存多套模板，设置默认模板

### 1.2 进销存管理模块
#### 1.2.1 采购入库管理
- **功能点1：采购单创建/编辑/提交/审核**
  - 仓库管理员/店长可创建采购单：录入供应商名称/联系方式、采购日期、预计到货日期、备注
  - 采购单商品录入：支持扫码/手动输入SKU唯一标识、输入采购数量、采购单价（必填）、折扣（可选，默认无）
  - 采购单提交后，需由超级管理员/店长审核，审核通过后方可入库；审核不通过需退回并填写原因
  - 支持草稿保存（未提交的采购单可随时编辑）
- **功能点2：采购入库操作**
  - 审核通过的采购单，仓库管理员可进行入库：核对实际到货数量（支持修改，但修改后需记录原因）、确认入库时间
  - 入库成功后，系统自动增加对应SKU的库存，生成采购入库流水记录
- **功能点3：采购单查询/导出**
  - 支持按采购日期、供应商、审核状态、入库状态筛选采购单
  - 支持导出Excel格式的采购单明细

#### 1.2.2 销售退货入库管理
- **功能点1：销售退货申请/审核**
  - 收银员/店长可创建销售退货单：录入原交易流水号（必填，系统自动关联原销售商品明细、会员信息、支付信息）、选择退货商品/数量、退货原因（必填，下拉选择+自定义补充）
  - 退货单提交后，需由超级管理员/店长审核（审核权限可在基础设置中调整预设角色），审核通过后方可退款入库；审核不通过需退回并填写原因
  - 支持限制退货时间（如基础设置中可配置“7天无理由退货”“30天质量问题退货”，超过时间的退货需店长以上级别审核才可提交）
- **功能点2：退款与入库操作**
  - 审核通过的退货单，仓库管理员先核对退货商品（确认是否符合退货条件），收银员再执行退款：按原支付方式原路退回（现金/会员卡余额支付可现场操作，第三方支付需调用接口发起退款）
  - 退款成功后，系统自动增加对应SKU的库存，生成销售退货流水记录，扣减对应会员的消费金额/积分
- **功能点3：退货单查询/导出**
  - 支持按退货日期、原交易流水号、审核状态、退款状态筛选退货单
  - 支持导出Excel格式的退货单明细

#### 1.2.3 单店内部调拨管理
- **功能点1：调拨单创建/审核/执行**
  - 仓库管理员/店长可创建调拨单：录入调出区域/货架、调入区域/货架、调拨日期、备注
  - 调拨单商品录入：支持扫码/手动输入SKU唯一标识、输入调拨数量
  - 调拨单提交后，需由超级管理员/店长审核，审核通过后方可执行
  - 执行调拨时，仓库管理员需再次核对商品/数量，确认后系统自动调整调出/调入区域的库存（总库存不变），生成调拨流水记录
- **功能点2：调拨单查询/导出**
  - 支持按调拨日期、调出/调入区域、审核状态、执行状态筛选调拨单
  - 支持导出Excel格式的调拨单明细

#### 1.2.4 库存盘点管理
- **功能点1：盘点任务创建/分配**
  - 超级管理员/店长可创建盘点任务：录入盘点日期、盘点区域/货架、分配盘点人（仓库管理员/店长）、备注
  - 支持创建“全店盘点”“区域盘点”“品类盘点”“SKU指定盘点”四种类型的任务
- **功能点2：盘点操作**
  - 盘点人可通过扫码枪/手动输入SKU唯一标识进行盘点，输入实际库存数量
  - 系统自动对比账面库存与实际库存，生成“盘盈”“盘亏”“账实相符”三种状态的明细
  - 支持草稿保存（未提交的盘点数据可随时编辑）
- **功能点3：盘点结果审核/调整**
  - 盘点人提交盘点结果后，需由超级管理员/店长审核，审核通过后方可调整库存；审核不通过需退回并填写原因
  - 调整库存时，系统自动生成盘盈/盘亏流水记录
- **功能点4：盘点单查询/导出**
  - 支持按盘点日期、盘点类型、审核状态、调整状态筛选盘点单
  - 支持导出Excel格式的盘点单明细（含盘盈/盘亏金额计算：盘盈金额=SKU吊牌价×盘盈数量，盘亏金额=SKU采购价×盘亏数量）

#### 1.2.5 库存预警管理
- **功能点1：预警阈值设置**
  - 支持按SKU/品类设置最低库存预警阈值、最高库存预警阈值
  - 最高库存预警阈值为可选设置
- **功能点2：预警提醒**
  - 系统实时监测库存，当SKU库存≤最低阈值时，在后台首页/仓库管理员/店长的消息中心推送“库存不足预警”
  - 当SKU库存≥最高阈值时，推送“库存积压预警”
  - 预警消息需展示SKU信息、当前库存、预警阈值、建议操作（如补货/促销）

### 1.3 POS收银管理模块
#### 1.3.1 POS收银台
- **功能点1：商品录入**
  - **扫码录入**：支持接入通用USB/蓝牙扫码枪，扫描商品吊牌上的SKU唯一标识，系统自动获取商品信息（SPU名称、SKU属性、吊牌价、当前库存）并加入购物车
  - **手动输入SKU**：若扫码失败，可手动输入SKU唯一标识或SPU名称后选择对应的SKU加入购物车
  - **多属性快速选品**：若仅知道SPU名称，可通过SPU名称搜索后，依次选择品类、颜色、尺码等属性快速生成SKU加入购物车
  - **库存校验**：加入购物车时自动校验当前库存，若库存不足则提示“库存不足，请调整数量或更换商品”
- **功能点2：购物车管理**
  - 支持修改购物车中商品的数量、删除商品
  - 支持添加商品备注（如“顾客需要包装成礼盒”）
  - 自动计算购物车商品的总吊牌价
- **功能点3：会员识别与权益应用**
  - **会员识别方式**：支持输入手机号、扫描会员码（可提前生成静态会员码或在后台查看会员动态码）识别会员
  - **权益自动应用**：识别会员后，系统自动展示会员等级、当前累计消费金额、当前可用积分、生日信息
  - **生日折扣自动触发**：若当天是会员生日（可在基础设置中配置“生日前后7天可享受生日折扣”），自动弹出“是否应用生日专属折扣（如8折）”的提示，确认后应用
  - **会员折扣自动应用**：若未应用生日专属折扣，自动应用会员等级对应的吊牌价折扣
  - **积分抵现**：支持手动输入抵现积分，系统自动根据基础设置中的“积分抵现规则”（如100积分抵1元）计算抵现金额，抵现金额不得超过当前可用积分对应的金额，也不得超过当前购物车商品的总应付金额（不含第三方支付手续费）
  - **权益叠加限制**：可在基础设置中配置“生日折扣与会员折扣不可叠加”“积分抵现可与任何折扣叠加”等规则
- **功能点4：优惠券/促销活动应用（可选补充，支持基础满减）**
  - 支持在基础设置中配置“满X元减Y元”“满X件打Y折”的单店通用促销活动
  - 加入购物车后，系统自动判断是否符合促销活动条件，自动应用最大优惠的活动（可手动切换活动）
  - 不支持自定义优惠券
- **功能点5：支付方式录入与结算**
  - **支持的支付类型**：需包含基础设置中启用的所有支付方式，默认按优先级排序（可在基础设置中调整优先级）
  - **现金支付**：输入顾客实付金额，系统自动计算找零金额，若实付金额小于应付金额则提示“实付金额不足”
  - **第三方支付（支付宝/微信支付）**：支持两种支付模式
    1. **扫码枪扫顾客付款码**：接入通用USB/蓝牙扫码枪，扫描顾客的支付宝/微信付款码，系统调用第三方支付接口发起扣款，扣款成功后自动生成支付记录
    2. **生成收款码供顾客扫描**：系统生成动态支付宝/微信收款码，展示在POS收银台，顾客扫描后完成付款，系统实时接收第三方支付的回调通知，确认付款成功后自动生成支付记录
  - **会员卡余额支付**：需先识别会员，输入支付金额（默认为当前应付金额），系统自动校验会员余额是否充足，若充足则扣款成功并生成支付记录，同时增加会员的累计消费金额、按会员等级积分倍数计算并增加积分；若不足则提示“会员卡余额不足，请切换支付方式”
  - **组合支付**：支持最多两种支付方式的组合（如会员卡余额支付+现金支付、支付宝支付+微信支付），需分别输入每种支付方式的金额，总金额需等于当前应付金额
  - **支付记录规则**：每笔交易的支付记录需包含以下信息：交易流水号、支付方式、支付金额、支付时间、支付状态（成功/失败/退款中/已退款）、第三方支付交易号（若有）、操作人
  - **支付记录关联规则**：每笔支付记录必须关联唯一的交易流水号，每笔交易流水号可关联1-2条支付记录（组合支付时），交易流水号必须关联小票打印记录
- **功能点6：小票打印与补打**
  - **即时打印**：支付成功后，系统自动调用默认的小票模板，通过连接的热敏小票打印机打印小票，支持打印1-3份（可在基础设置中设置默认打印份数）
  - **手动补打**：可在POS收银台的“历史交易”中查询交易流水号，点击“补打”按钮打印小票，补打小票上需标注“补打”字样及补打时间
- **功能点7：历史交易查询**
  - 支持在POS收银台查询当天的所有交易流水，支持按交易时间、收银员、支付方式、会员手机号筛选
  - 支持查看交易明细（商品明细、优惠明细、支付明细）
- **功能点8：POS交接班管理**
  - 收银员交接班时，可点击“交接班”按钮，系统自动统计当前班次的销售总金额、各支付方式的金额、现金找零备用金的差额（备用金可在基础设置中设置，需手动输入当前备用金余额）
  - 生成交接班报表，支持打印，交接班报表需经下一班收银员/店长确认后生效

### 1.4 流水报表管理模块
#### 1.4.1 销售流水报表
- **支持的筛选维度**：日期（日/周/月/季/自定义时间段）、品类、SKU属性（颜色/尺码/材质）、收银员、会员、支付方式、销售状态（正常销售/已退货/部分退货）
- **展示的核心指标**：销售总金额、销售总成本、销售总利润、销售总数量、客单价
- **展示的明细内容**：交易流水号、交易时间、收银员、会员信息、商品明细（SPU名称、SKU属性、数量、吊牌价、折扣后单价、小计）、优惠明细、支付明细、利润小计
- **导出功能**：支持导出Excel格式的销售流水报表（含核心指标汇总表+明细表）

#### 1.4.2 采购流水报表
- **支持的筛选维度**：日期（日/周/月/季/自定义时间段）、供应商、品类、SKU属性、审核状态、入库状态
- **展示的核心指标**：采购总数量、采购总金额
- **展示的明细内容**：采购单号、采购日期、供应商信息、商品明细（SPU名称、SKU属性、数量、采购单价、折扣后单价、小计）、审核人、审核时间、入库人、入库时间
- **导出功能**：支持导出Excel格式的采购流水报表（含核心指标汇总表+明细表）

#### 1.4.3 库存流水报表
- **支持的筛选维度**：日期（日/周/月/季/自定义时间段）、品类、SKU属性、流水类型（采购入库/销售退货入库/内部调拨/盘盈/盘亏/销售出库）
- **展示的明细内容**：流水号、流水时间、流水类型、商品明细（SPU名称、SKU属性、数量、单价、小计）、操作人、备注
- **导出功能**：支持导出Excel格式的库存流水报表

#### 1.4.4 会员消费报表
- **支持的筛选维度**：日期（日/周/月/季/自定义时间段）、会员等级、会员性别、会员年龄区间
- **展示的核心指标**：会员总人数、活跃会员人数（自定义时间段内有消费的会员）、会员消费总金额、会员消费总利润、会员人均消费金额、会员复购率
- **展示的明细内容**：会员姓名、会员手机号、会员等级、累计消费金额、累计消费次数、自定义时间段内的消费金额/次数、可用积分
- **导出功能**：支持导出Excel格式的会员消费报表（含核心指标汇总表+明细表）

#### 1.4.5 收银员业绩报表
- **支持的筛选维度**：日期（日/周/月/季/自定义时间段）、收银员
- **展示的核心指标**：收银员销售总金额、收银员销售总利润、收银员销售总数量、收银员客单价、收银员接待会员人数
- **展示的明细内容**：交易流水号、交易时间、收银员、商品明细、利润小计
- **导出功能**：支持导出Excel格式的收银员业绩报表（含核心指标汇总表+明细表）

#### 1.4.6 利润分析报表
- **支持的筛选维度**：日期（日/周/月/季/自定义时间段）、品类、SKU属性
- **展示的核心指标**：总销售额、总采购成本、总运营成本（需手动录入后台，可选功能）、总利润、毛利率
- **展示的明细内容**：按品类/SKU属性分组的销售额、采购成本、利润、毛利率
- **导出功能**：支持导出Excel格式的利润分析报表（含核心指标汇总表+分组明细表）

## 2. 非功能需求
### 2.1 性能需求
1. **响应时间**：
   - 后台首页加载时间≤2秒
   - POS收银台商品录入（扫码/手动输入）响应时间≤0.5秒
   - 单份复杂报表（含10000条以上明细）生成时间≤3秒
   - 第三方支付接口调用响应时间≤5秒（超时需提示“支付超时，请稍后查询交易状态”）
2. **并发处理**：
   - 支持最多5个用户同时在线（其中最多1个用户使用POS收银台）
3. **数据存储**：
   - 支持存储至少5年的业务数据（含流水记录、SPU/SKU信息、会员信息）
   - 支持数据备份功能：每天凌晨自动备份数据库，备份文件保留30天；支持手动备份数据库

### 2.2 安全需求
1. **身份认证**：
   - 所有用户必须通过用户名+密码登录系统，密码需加密存储（采用SHA-256+盐值加密）
   - 支持设置密码强度要求（如至少8位，含大小写字母+数字+特殊字符）
   - 支持设置登录超时时间（如30分钟无操作自动退出登录）
2. **权限控制**：
   - 严格按照基础设置中的角色权限控制用户的操作，未授权的功能模块/操作按钮对用户不可见
   - 所有涉及数据修改/删除/审核的操作需记录操作日志（操作人、操作时间、操作前数据、操作后数据、操作内容），操作日志不可删除，仅超级管理员可查看
3. **数据安全**：
   - 支持HTTPS协议传输数据（部署时需配置SSL证书）
   - 第三方支付接口调用需采用官方SDK，确保支付数据的安全性
   - 会员手机号等敏感信息需加密存储

## 4. 技术约束
### 4.1 技术栈
- **前端框架**：Vue.js 3.x + Element Plus（适配PC端Web页面，POS收银台需适配大字体/大按钮的操作界面）
- **后端框架**：Spring Boot 2.7.x（稳定可靠，适合单店独立部署）
- **数据库**：MySQL 8.0.x（关系型数据库，适合存储结构化的业务数据）
- **第三方支付SDK**：支付宝官方Java SDK、微信支付官方Java SDK（V3版本）
- **打印插件**：lodop（通用热敏小票打印机打印插件，支持Windows/Mac系统）
- **部署工具**：Docker（简化单店独立部署流程）

### 4.2 平台要求
- **客户端**：支持Windows 10及以上、Mac OS 11及以上系统的Chrome浏览器（最新版）、Edge浏览器（最新版）
- **服务器**：支持Windows Server 2016及以上、Linux CentOS 7及以上系统的服务器（单店建议配置：4核CPU、8G内存、100G SSD硬盘）

### 4.3 集成要求
- **扫码枪集成**：支持接入所有通用的USB/蓝牙扫码枪（无需安装额外驱动，浏览器可直接识别）
- **热敏小票打印机集成**：支持接入所有通用的USB/蓝牙热敏小票打印机（需安装lodop插件）
- **第三方支付集成**：需提供支付宝商户号/API密钥、微信支付商户号/API密钥/V3密钥/商户证书，系统需调用官方SDK完成支付/退款/回调通知的功能

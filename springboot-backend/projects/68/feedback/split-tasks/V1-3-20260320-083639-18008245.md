# 童趣衣橱3-12岁儿童服装单店进销存POS管理系统技术任务书
## 1. 项目技术规格
- 技术栈：
  - 前端：Vue.js 3.4.21 + Element Plus 2.6.3 + Vite 5.2.8 + Pinia 2.1.7 + Axios 1.6.8
  - 后端：Spring Boot 2.7.18 + MyBatis Plus 3.5.5 + MySQL 8.0.36 + JWT 0.11.5 + Hibernate Validator 6.2.5.Final
  - 第三方服务：支付宝Java SDK 4.39.70.ALL、微信支付Java SDK 0.2.12、lodop 6.5.7.0
  - 部署：Docker 24.0.7 + Docker Compose 2.21.0
- 项目结构规范：
  - 后端（kidswear-pos-backend）：
    - src/main/java/com/kidswear/pos/：controller、service、serviceImpl、mapper、entity、dto、vo、config、utils、interceptor、exception
    - src/main/resources/：application.yml、application-dev.yml、application-prod.yml、db/init.sql、db/test_data.sql
    - src/test/java/：单元测试目录
  - 前端（kidswear-pos-frontend）：
    - src/：views/（基础设置、进销存、POS、报表）、components/（public/、business/）、api/、router/、store/、utils/、assets/
    - index.html、package.json、vite.config.ts
- 代码生成规则：
  - 依赖版本统一用技术栈指定版本
  - 数据校验：实体类使用JSR380注解，DTO/VO加自定义业务校验
  - JWT配置：过期时间30分钟，强随机Base64密钥，前端存储在localStorage，拦截器验证
  - 敏感数据加密：密码SHA-256+盐值，会员手机号AES-128加密
  - 打印规则：默认调用lodop打印默认模板，支持切换模板和补打标记
- 核心业务领域锚点：【童趣衣橱3-12岁儿童服装单店进销存POS管理系统】

## 8. 补充接口规范（按常规任务书完整性添加）
### 接口通用约束
1. **通信协议**：HTTP/HTTPS，生产环境必须使用HTTPS
2. **数据格式**：请求参数（除文件上传外）使用JSON格式，响应参数统一使用JSON格式
3. **字符编码**：统一使用UTF-8
4. **分页参数**：分页查询必须包含pageNum（页码，默认1）、pageSize（每页条数，默认20，最大1000）
5. **日期格式**：统一使用yyyy-MM-dd HH:mm:ss（时间戳或LocalDateTime转换）
6. **金额格式**：统一使用DECIMAL(10,2)，避免浮点数精度问题
7. **身份认证**：所有非登录接口必须在Header中携带Authorization: Bearer {token}
8. **权限控制**：所有接口必须在方法上添加@PreAuthorize("hasAuthority('权限编码')")注解
9. **操作日志**：所有涉及数据修改/删除/审核的接口必须自动记录操作日志

### 响应通用格式
```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {}
}
```
- **code**：状态码，200表示成功，其他表示失败（参考错误码定义）
- **msg**：提示信息，成功时为“操作成功”，失败时为具体错误信息
- **data**：响应数据，成功时返回业务数据，失败时返回null或错误详情

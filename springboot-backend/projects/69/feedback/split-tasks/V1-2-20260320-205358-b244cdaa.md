## 1. 项目技术规格
- 技术栈：
  前端：React 18.x + Ant Design 5.x（店长端）/ Vant 4.x（客户端） + Redux Toolkit（店长端）/ Zustand（客户端） + React Router 6.x + Vite 4.x + ESLint + Prettier
  后端：Node.js 18.x + NestJS 9.x + MySQL 8.x + TypeORM 0.3.x + Redis 7.x + Swagger/OpenAPI 3.0 + ESLint + Prettier + JWT
  第三方服务：阿里云短信服务、阿里云OSS、微信JSAPI/Native支付、支付宝手机/电脑网站支付
  部署：Docker + Nginx + Let's Encrypt SSL + Prometheus + Grafana

- 代码生成规则：
  依赖版本：严格遵循技术栈指定版本
  校验规则：前端Formik/Zod/ClassValidator，后端ClassValidator/ClassTransformer
  JWT配置：access_token有效期2小时，refresh_token有效期7天，密钥为环境变量，店长端/客户端权限用不同claims区分
  文件上传：图片前端压缩≤500KB（主图轮播）/≤200KB（详情），视频前端转码MP4≤720P≤2Mbps≤30s≤100MB，断点续传支持
  短信发送：每分钟同一手机号最多1次，验证码5分钟失效，使用后立即删除
  支付回调：严格签名验证+来源IP验证，支付状态30分钟内重试3次查询
- 核心业务领域锚点：【童趣小筑儿童服装进销存&线上商城】

## 5. 业务逻辑规则
### 核心业务流程
1. **儿童服装采购入库流程**：店长创建采购入库单→选择供应商（可选）、入库日期→添加SKU（扫码/手动）、修改进价（可选）、填写数量→提交→系统校验SKU存在/数量≥0→更新SKU库存→生成入库操作日志→结束
2. **儿童服装线上订单流程**：会员浏览商品→选择SKU→加购/立即购买→结算→选择配送方式/自提时间/收货地址→选择优惠券→提交订单→系统校验SKU库存/配送范围/自提时间/优惠券→锁定库存→生成订单号→跳转到第三方支付→支付成功→系统更新订单状态为待发货/待自提→自动生成销售出库单（预留）→发送支付成功短信→店长处理发货/自提核销→会员确认收货→订单完成→自动更新会员累计消费/等级→结束
3. **儿童服装库存盘点流程**：店长创建盘点单→选择盘点范围→生成盘点商品→SKU状态设为盘点中→执行盘点（扫码/手动输入实际库存）→暂停/继续→完成盘点→系统生成差异表→店长确认差异→更新SKU库存→SKU状态设为正常→恢复出入库→生成盘点操作日志→结束
4. **儿童服装会员自动升级流程**：会员完成订单支付→系统计算该会员累计消费金额/次数→对比系统配置的会员等级规则→若满足更高等级→自动升级该会员等级→发送升级成功短信（可选）→结束

### 数据校验规则
1. **儿童服装SKU校验**：
   - SKU编码唯一，格式支持字母+数字（手动/自动生成）
   - 尺码必须为系统配置的或自定义添加的有效儿童尺码
   - 库存数量≥0
   - 进价、售价≥0.01元
   - 最低预警值<最高预警值（若设置单个）
2. **儿童服装订单校验**：
   - 本地配送地址必须在系统配置的配送范围内（以门店地址为圆心的直线距离，预留调用地图API）
   - 自提时间必须在系统配置的自提时间范围内，且≥当前时间+30分钟
   - 优惠券使用范围覆盖勾选的所有SKU，使用门槛≤勾选SKU会员折扣小计之和，会员等级符合限制，未过期，未使用，剩余数量充足
   - 实付金额=商品总金额-会员折扣金额-优惠券折扣金额+运费≥0.01元
3. **儿童服装会员校验**：
   - 手机号唯一，格式为中国大陆11位有效手机号
   - 收货地址数量≤5个
4. **系统配置校验**：
   - 会员等级最多5级
   - 配送范围半径1-10公里
   - 手机验证码有效时间1-10分钟，每分钟同一手机号最多1-5次

### 状态流转规则
1. **儿童服装订单状态流转**：
   - pending-payment（待支付）→【支付成功】pending-ship-pickup（待发货/待自提）→【本地配送发货】shipped-pickup-confirm（已发货/待自提确认）→【确认送达/7天后自动确认】completed（已完成）
   - pending-payment（待支付）→【30分钟未支付/手动取消】cancelled（已取消）
   - pending-ship-pickup（待自提）→【自提核销】completed（已完成）
   - pending-ship-pickup（待自提）→【7天未自提】cancelled（已取消）
   - completed（已完成）→【7天内申请退款/售后】refunding/refunded（退款中/已退款）
2. **儿童服装退款/售后状态流转**：
   - pending-audit（待审核）→【24小时内审核通过】pending-return（退货退款）/refunding（仅退款）
   - pending-audit（待审核）→【24小时内审核拒绝】audit-rejected（审核拒绝）
   - pending-return（待退货）→【7天内会员提交退货】returned（已退货）
   - returned（已退货）→【店长确认收货】refunding（退款中）
   - returned（已退货）→【店长拒绝收货】return-rejected（退货拒绝）
   - refunding（退款中）→【第三方退款成功】refunded（已退款）

### 异常处理规则
1. **支付异常**：
   - 支付回调超时或失败：系统自动在30分钟内重试3次查询支付状态
   - 支付金额与订单实付金额不一致：系统标记订单为异常，通知店长手动处理
2. **短信异常**：
   - 短信发送失败：系统记录失败日志，支持店长在后台手动重发
3. **库存异常**：
   - 加购/提交订单时SKU库存不足：系统提示会员该SKU已售罄或库存不足
   - 出库/盘点时SKU库存<0：系统标记该SKU为异常库存，通知店长手动处理

## 7. 代码生成规范
### 命名规范
1. **类名**：大驼峰命名法，必须包含童趣小筑或对应业务模块关键词，例如：
   - 后端：TongquAuthController、ClothingSkuService、MemberLevelEntity
   - 前端：AdminLoginPage、ClothingSkuTable、ClientMallHome
2. **方法名**：小驼峰命名法，动词开头，例如：getClothingSkuList、createPurchaseInbound
3. **变量名**：小驼峰命名法，清晰表达含义，例如：memberNo、skuCode、totalPayAmount
4. **表名**：全小写，下划线分隔，前缀tq_，例如：tq_clothing_sku、tq_member_coupon
5. **字段名**：全小写，下划线分隔，清晰表达含义，例如：member_no、sku_code

### 错误码定义
错误码格式：TQ_模块缩写_00X（模块缩写：SYSTEM系统、CLOTHING服装、MEMBER会员、COUPON优惠券、ORDER订单、REFUND退款、MALL商城、AUTH认证），例如：
- TQ_AUTH_001：账号或密码错误
- TQ_CLOTHING_001：服装SKU不存在
- TQ_ORDER_001：订单不存在
- TQ_REFUND_001：退款/售后申请不存在

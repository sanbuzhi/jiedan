## 1. 项目技术规格
### 技术栈
- **前端技术栈**：
  店长端：React 18.x + Ant Design 5.x + Redux Toolkit 2.x + React Router 6.x + Vite 4.x + ESLint + Prettier
  客户端：React 18.x + Vant 4.x + Zustand 4.x + React Router 6.x + Vite 4.x + ESLint + Prettier
- **后端技术栈**：
  Node.js 18.x + NestJS 9.x + TypeORM 0.3.x + MySQL 8.x + Redis 7.x + class-validator + class-transformer + JWT
- **第三方集成**：
  微信JSAPI/Native支付、支付宝手机/电脑网站支付、阿里云短信服务、阿里云OSS
- **部署技术**：
  Docker + Docker Compose + Nginx + Let's Encrypt SSL证书 + Prometheus + Grafana

### 代码生成规则
- **依赖版本**：严格按照技术栈指定的大版本锁定（使用npm shrinkwrap或yarn.lock）
- **校验规则**：
  - 后端：使用class-validator校验DTO参数，自定义儿童服装相关校验（如尺码格式、效期合法性、价格非负等）
  - 前端：使用AntD/Vant内置表单校验+自定义正则表达式
- **JWT配置**：
  - 访问令牌（access_token）：店长端有效期7天，客户端有效期3天，算法HS256
  - 刷新令牌（refresh_token）：店长端有效期30天，客户端有效期15天，仅用于刷新访问令牌
  - 密钥：使用随机生成的32位以上字符串存储在环境变量中
- **文件上传规则**：
  - 商品图片：主图/轮播图≤500KB（压缩至750×750/750×1000px），详情图≤200KB，格式仅支持JPG、PNG、WEBP
  - 商品视频：≤100MB，转码为MP4 720P 2Mbps，时长≤30秒
  - 临时文件（Excel导入导出等）：存储在OSS临时目录，有效期24小时

## 4. 数据库表结构设计
### 核心业务表
#### tongqu_user（用户表）
| 字段名         | 数据类型    | 约束        | 默认值      | 注释                      |
|----------------|-------------|-------------|-------------|---------------------------|
| id             | BIGINT      | PRIMARY KEY, AUTO_INCREMENT | 无          | 用户ID                    |
| identity       | VARCHAR(20) | NOT NULL    | 'client'    | 身份：admin/client        |
| phone          | VARCHAR(20) | UNIQUE, NOT NULL | 无          | 手机号（唯一标识）        |

## 5. 业务逻辑规则
### 核心业务流程
#### 1. 儿童服装采购入库流程
1. 店长在入库管理模块创建「采购入库单」，选择商品SKU、填写入库数量和进价（默认使用SKU当前进价）
2. 提交入库单前可保存为草稿并多次修改，提交后系统自动校验库存状态（非盘点中）、SKU状态（上架/下架均可入库）
3. 校验通过后，更新对应商品SKU的可用库存，生成入库操作日志
4. 自动检查库存是否超过全局或SKU单独设置的最高预警值，若超过则生成滞销预警消息并推送到店长端首页

#### 2. 儿童服装线上订单流程
1. 客户在客户端浏览商品，选择SKU后加购或立即购买
2. 结算时：
   - 选择配送方式（本地配送/自提，本地配送需检查地址是否在配置的配送范围内）
   - 若选择自提，需选择自提时间（默认当前时间+30分钟，且在门店自提时间范围内）
   - 若选择本地配送，需选择收货地址（默认使用默认收货地址）
   - 系统自动筛选并展示符合条件的优惠券（可选择1张使用）
   - 系统自动计算商品总金额、会员折扣、优惠券优惠、运费、实付金额
3. 提交订单后：
   - 系统自动生成唯一订单号
   - 锁定对应商品SKU的可用库存（从可用库存转移到锁定库存）
   - 若选择自提，生成6位自提验证码并存储到Redis（有效期至自提时间后1小时）
   - 订单状态变为「待支付」，客户需在30分钟内完成支付
4. 支付处理：
   - 客户选择支付方式（微信/支付宝），跳转到第三方支付页面
   - 支付成功后，第三方平台发送回调，系统校验回调签名后更新订单状态为「待发货/待自提」，释放锁定库存并扣减可用库存，更新会员累计消费金额和次数，自动检查是否升级会员等级，发送支付成功短信
   - 支付超时或客户取消订单，系统自动释放锁定库存，订单状态变为「已取消」，发送取消订单短信

#### 3. 会员优惠券发放核销流程
1. 店长创建优惠券，设置发放规则、使用规则、有效期，提交后优惠券状态变为「未开始」或「进行中」（根据发放时间判断）
2. 优惠券发放：
   - 手动发放：选择发放对象（单个会员/按会员等级/按注册时间/按累计消费金额），发放后发送发放成功短信
   - 自动发放：系统根据预设的触发条件（会员注册成功、会员生日当天、会员达到某个等级、会员累计消费金额达到某个阈值）自动发放，发放后发送发放成功短信
3. 优惠券核销：
   - 客户在结算时选择符合条件的优惠券，系统自动校验使用范围、使用门槛、会员等级限制、有效期、是否已使用、每人限领数量
   - 校验通过后，更新优惠券使用状态为「已使用」，更新优惠券剩余发放数量（若有），生成核销操作日志

## 7. 代码生成规范
### 命名规范
#### 1. 类名/接口名/枚举名
- 采用大驼峰命名法（PascalCase）
- 必须包含核心业务领域标识「Tongqu」
- 示例：TongquUserController、TongquProductSkuService、TongquOrderStatusEnum、ITongquCouponService

#### 2. 方法名/变量名
- 采用小驼峰命名法（camelCase）
- 尽量使用儿童服装相关的业务词汇
- 示例：createChildClothingSku、getTongquMemberList、stockInQuantity、childClothingSize、selfPickupCode

#### 3. 表名/字段名
- 采用小写下划线命名法（snake_case）
- 表名前缀必须为「tongqu_」
- 尽量使用儿童服装相关的业务词汇
- 示例：tongqu_product_sku、tongqu_member_coupon、child_clothing_size、self_pickup_code

#### 4. 接口URL
- 采用小写下划线命名法（snake_case）
- 前缀必须为「/api/tongqu/」
- 店长端接口前缀为「/api/tongqu/admin/」，客户端接口前缀为「/api/tongqu/client/」
- 尽量使用RESTful风格
- 示例：/api/tongqu/admin/inventory/sku/add、/api/tongqu/client/product/detail/:id、/api/tongqu/admin/member/coupon/send

### 错误码定义
- 错误码格式为「TQ_模块码_错误码」，其中：
  - 前缀「TQ」为核心业务领域标识
  - 中间三位为模块标识：
    - 001：公共模块
    - 002：店长端系统基础模块
    - 003：店长端进销存管理模块
    - 004：店长端会员管理模块
    - 005：店长端订单管理模块
    - 006：店长端数据报表模块
    - 007：客户端模块
  - 最后三位为具体错误标识，从001开始递增
- 示例：
  - TQ_001_001：参数校验失败
  - TQ_001_002：身份认证失败
  - TQ_001_003：权限不足
  - TQ_003_001：商品SKU不存在
  - TQ_003_002：儿童服装库存不足
  - TQ_003_003：正在盘点中，无法进行库存操作
  - TQ_005_001：订单不存在
  - TQ_005_002：订单状态不正确，无法进行该操作
  - TQ_007_001：收货地址不在配送范围内

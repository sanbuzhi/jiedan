### 代码生成规则
1. 依赖版本严格使用上述技术栈中指定的稳定版
2. 数据校验使用Hibernate Validator，覆盖所有DTO参数
3. JWT配置：密钥为`kid_children_clothing_20240501`，算法HS256，有效期7天，刷新阈值为剩余1天内自动刷新
4. 数据库字符集：`utf8mb4`，排序规则`utf8mb4_general_ci`
5. 时间格式化统一使用`yyyy-MM-dd HH:mm:ss`
6. 金额统一使用BigDecimal类型，保留2位小数，四舍五入

## 5. 业务逻辑规则
### 核心业务流程
#### 1. 儿童服装SKU上下架+库存同步流程
1. **SKU创建/调整**：新增商品或下架后设置SKU结构（颜色、尺码），生成所有属性组合，填写库存、成本价、条码
2. **预警阈值设置**：自动/手动为SKU设置最低/最高库存预警阈值
3. **上下架验证**：上架时验证商品是否有至少1个SKU有库存
4. **库存同步**：入库/出库/盘点/销售发货/退货退款时实时调整对应SKU的库存，重新触发预警检查
5. **线上展示同步**：上下架/库存变动后线上商城实时更新商品可见性和SKU库存状态（已售罄/有货）

#### 2. 全渠道订单统一处理流程
1. **订单创建**：
   - 线上：客户提交订单后生成MALL开头的订单，状态「待付款」，锁定购物车项，设置30分钟过期时间
   - 线下：店长录入订单后生成STORE开头的订单，状态「已付款」，直接生成销售出库单
2. **订单支付**：
   - 线上：客户选择支付方式（微信/支付宝/储值），调用支付接口，回调验证通过后更新状态为「已付款」，解锁购物车项
   - 线下：直接选择支付方式，若用储值则扣除储值余额
3. **订单发货**：
   - 线上：店长在后台选择物流、填写运单号，更新状态为「已发货」，自动生成销售出库单，设置7天自动确认收货时间
   - 线下：直接生成销售出库单
4. **订单确认收货**：
   - 线上：客户手动确认或7天后自动确认，更新状态为「已完成」，积分（若有）实时到账
5. **订单退款**：
   - 线上：客户或店长申请退款，更新状态为「退款中」，店长审核通过后生成退款单，积分、优惠券、储值余额原路返还，自动生成退货入库单，更新状态为「已退款」

#### 3. 会员等级自动调整流程
1. **定时任务触发**：每日凌晨2点触发定时任务
2. **数据计算**：遍历所有非删除会员，计算累计消费金额/次数
3. **等级匹配**：根据生效中的会员等级规则，从低到高匹配会员可升级到的最高等级
4. **等级升级**：若可升级，则更新会员等级，发放专属优惠券（若有），记录操作日志
5. **不降级**：仅支持等级升级，不支持自动降级

#### 4. 儿童服装库存盘点流程
1. **创建盘点单**：店长选择盘点范围（全仓/指定分类/标签/SKU），生成盘点单，状态「待盘点」
2. **录入盘点数据**：店长通过扫码枪或手动录入每个SKU的实际库存
3. **确认盘点结果**：店长核对系统库存与实际库存的差异，二次确认后更新状态为「已完成」
4. **库存调整**：系统自动生成「盘点差异入库单」或「盘点差异出库单」，调整对应SKU的库存至实际库存，重新触发预警检查

### 数据校验规则
#### 通用校验
1. 手机号：中国大陆11位有效手机号（正则：`^1[3-9]\d{9}$`）
2. 密码：B2S管理后台8-20位大小写字母/数字/特殊字符组合（正则：`^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]{8,20}$`）
3. 金额：BigDecimal类型，保留2位小数，四舍五入，≥0.01元（除吊牌价/销售价对比、积分抵扣等场景）
4. 数量：INT类型，≥0整数（除购买/入库/出库等场景≥1）
5. 图片：JPG/PNG格式，≤5MB，宽高比1:1（主图/Logo/Avatar），详情图≤10张
6. 文本：商品名称2-50字符，昵称/真实姓名/收货人≤20字符，商品描述≤5000字符，备注≤200字符，XSS过滤后存储

#### 儿童服装特定校验
1. **童装预设尺码**：幼童装/中大童装预设尺码为「59」「66」「73」「80」「90」「100」「110」「120」「130」「140」「150」「160」，支持自定义
2. **SKU库存验证**：手工出库/订单发货/购买数量不得超过对应SKU的当前库存
3. **已上架商品修改限制**：已上架商品可修改除「商品分类」「SKU结构」外的所有信息
4. **商品删除限制**：仅允许删除已下架且无任何库存、订单记录的商品

### 状态流转规则
#### B2S管理后台商品状态流转
```
下架(0) → 上架(1) [需至少1个SKU有库存]
上架(1) → 下架(0) [无限制]
```

#### B2S管理后台库存预警状态流转
```
无预警 → 缺货预警(1) [SKU当前库存≤最低库存]
无预警 → 积压预警(2) [设置了最高库存且SKU当前库存≥最高库存]
缺货预警(1)/积压预警(2) → 已处理(0) [店长标记]
```

#### B2C线上订单状态流转
```
待付款(1) → 已付款(2) [客户支付]
待付款(1) → 已取消(5) [客户/店长取消/30分钟自动取消]
已付款(2) → 待发货(2？修正：线上状态1待2付3待发4已发5完6退中7退)
已付款(3？线上修正后统一状态：1待付款2已付款3待发货4已发货5已完成6已取消7退款中8已退款)
已付款(2) → 待发货(3) [支付完成后自动]
待发货(3) → 已发货(4) [店长发货]
已发货(4) → 已完成(5) [客户手动确认/7天自动确认]
已发货(4) → 退款中(7) [客户/店长申请]
退款中(7) → 已退款(8) [店长审核通过]
退款中(7) → 已完成(5) [店长审核拒绝]
```

#### B2S线下订单状态流转
```
待付款(1) → 已付款(2) [店长录入支付]
待付款(1) → 已取消(3) [店长取消]
已付款(2) → 已完成(4) [发货后自动？线下无需发货，直接已完成]
```

### 异常处理规则
#### 通用异常
1. **参数校验异常**：返回`KIDS_CLOTHING_001`错误码，提示具体参数错误原因
2. **未授权异常**：返回`KIDS_CLOTHING_002`错误码，401状态码，跳转登录页
3. **数据不存在异常**：返回`KIDS_CLOTHING_003`错误码，提示数据不存在
4. **业务逻辑异常**：返回`KIDS_CLOTHING_004`错误码，提示具体业务错误原因
5. **系统异常**：返回`KIDS_CLOTHING_005`错误码，提示系统繁忙，请稍后再试，记录错误日志

#### 儿童服装特定异常
1. **SKU库存不足异常**：返回`KIDS_CLOTHING_101`错误码，提示SKU库存不足
2. **已上架商品不可修改分类/SKU结构异常**：返回`KIDS_CLOTHING_102`错误码，提示需先下架商品
3. **商品不可删除异常**：返回`KIDS_CLOTHING_103`错误码，提示需先下架且清空库存和订单记录
4. **优惠券不可用异常**：返回`KIDS_CLOTHING_201`错误码，提示优惠券不可用的具体原因（过期/已使用/不满足使用门槛/不适用范围）
5. **储值余额不足异常**：返回`KIDS_CLOTHING_202`错误码，提示需先储值或切换其他支付方式
